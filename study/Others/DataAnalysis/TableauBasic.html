Tableau とは

tableau は CSV や Excel などのデータソースから読み込んだデータを可視化することで、視覚的なデータ分析をサポートします。ドキュメントはこちらのサイト(外部リンク)にありますが、難読なのでこちらのページで簡略化して基礎的な使い方などを紹介します。

データソースの用語

データソースとは、tableau が読み込むデータが保管されたファイルやデータベースを指します。基本的にデータベースなどは以下のようなテーブルでデータを管理します。

第 \(1\) 行 (第 \(0\) 行) に書き込まれている要素をヘッダーと呼び、続く行が持つデータの属性の命名の役割を持ちます。

記録は行単位で行われ、\(1\) 行に対応するデータの組をレコードと呼びます。異なるレコード同士に関係性を定義することはなく、各行は独立な記録を表します。レコードを記録する際は、レコードの属性を表すデータと各列のヘッダーの命名が対応するように書き込む必要があります。

赤枠で囲まれたレコードは、2024/5/18 の 22:40 に終了したプログラミングコンテストの結果を記録したレコードです。この枠内の各セルに記録されたデータは、そのセルの列のヘッダーが命名する属性に対応しています。

余談ですが、レコードを識別するために使用する Key (属性,列名) のことを主要キー ないし Primary Key と呼びます。Primary Key となるデータの値は全てのレコードでユニークである (重複しない) 必要があります。例えば、以下のテーブルのコンテストという列は Primary Key の一つです。

データソース

tableau ファイルで表示するデータの参照先をデータソースと言います。データソースを参照できる状態にすることを接続と呼び、主要な接続方法は以下の \(2\) 種類です。

  1. 抽出 : データソースを tableau ファイルに統合して、同じファイル内で接続する状態
  2. ライブ : 外部のデータソースを常に参照する状態
どちらにするべきとかはありませんが、「接続が~」みたいなメッセージが出たらここを思い出すと良いです。

データソースの選択

さまざまな形式のファイルをデータソースとして読み込むことができますが、全てのデータはテーブル形式に変換されます。例えば CSV を読み込むと、CSV の第 \(1\) 行 (第 \(0\) 行) の要素がヘッダーとして登録され、以降の行がレコードとして記録されます。また、json ファイルではキーがヘッダーに登録され、各キーに対する値がレコードに記録されます。

ビューの作成

ビューとは ~

ビューのデータの表示に関して、データを以下の \(3\) の概念に分けて考える必要があります。

  • ディメンジョン : レコードの分類を指定するデータ要素
  • 属性 : 対象のデータ(マーク)のラベル
  • 集計 : ディメンジョンによる各分類について、ある分類に属するレコードのデータの集計
