そうしてできる文字列を T と一致させるためには必要な x の最小値を求める。

解法 :
f( x ) := S の先頭から x 個取り除くとき、そこから操作することで T に一致させることができるか

ARC144-B


スコアを X 以上にできるかと言う判定を考えると、X を 「できる領域」と「できない領域」に二分できる。
よって二分探索してみよう。

A[ i ] の値が X 以上であれば、A[ i ] に何回 -b できるかをカウントする。
A[ i ] の値が X 未満であれば、A[ i ] に何回 +a する必要があるかをカウントする。
それぞれのカウントは 0( 1 ) ででき、 +a のカウントが -b のカウント以下であれば良い。
判定が O( N ) でできたので、二分探索することで X の最大値がもとまる。



ARC140-A


f( S ) は、 S の最短周期の長さがいくつかということである。
つまり、S を 長さ L の文字列が N/L 個連結したものと考えると、操作を k 回以下行った上でそれらの文字列全てを同じにできるような L を最小化する問題と言い換えれる。

L の候補は、N の約数のみである。

S を N/L 個の文字列に分けた時、
それら N/L 個の文字列が全て同じになるような操作回数は、
各領域の x 文字目を全て同じ文字にするために必要な操作回数
を x についての足し合わせたものである。これが k 以下であれば良い。

N は 2000以下、L は N の約数全てなので、愚直にしても間に合う。



ARC140-B


文字列 S の連続部分文字列で、AA..A R C..CC となっている部分、つまり x 個の A と 1 個の R と x 個の C が順に並んでいる部分を全て取り出す。
なお、それらの部分文字列は重複する区間を持たないことがわかる。

取り出した区間たちの長さを記録した数列を S とする。
取り出した部分文字列を１つ選び、偶数回目の操作を行うと、操作を行った部分文字列にはそれ以降操作を行えなくなる。
よって、長さが長い部分文字列に対して偶数回目の操作を行うことはもったいないことがわかる。

以上から、操作回数を最大化するためには、
奇数番目の操作は、長さが最大の部分文字列に対して行い、
偶数回目の操作は、長さが最小の部分文字列に対して行う
(そして、偶数回目の操作を行った文字列はSからpopする)。
これを操作が行えなくなるまで行えば良い。

平衡二分木を使うと、最大/最小を高速に取り出した上で、push、popもできる。

街 A[ i ] と A[ i + 2 ] の間に辺を貼る場合のコストは、A[ i ] と A[ i + 1 ] と A[ i + 2 ]の3つの街に 2 つの辺を貼ってつなげる場合のコストと等しい。

潜入


ダイクストラだが、4 つ目の操作が曲者。
2 つの世界を移動することを考える。
1 ~ 3 の操作は 1 つ目の世界を、各移動毎に、決められたコストを払って移動するとして、
4 つ目の操作を行ったら、コストを 1 払って 2 つ目の世界に移動することとする。

2 つ目の世界では、上に 1 移動するたびに コストが 1 かかり、任意の座標から1 つ目の世界にコスト 0 で戻ってこれることとする。

こうすることで、 4 つ目の操作で上に i 移動した際のコストは、
・1 払って 2 つ目の世界に移動する.
・i 払って 2 つ目の世界で上に i 移動する。
となるので、見事 i + 1 コストで 4 つ目の操作を実現できました。



Potion


K 個選んで作る時、毎秒魔力が K 増加する。
K は定数なので、K 個選んで作ったポーションの初期の魔力が m とした時、X - m が K の倍数でなければならない。

DP 配列を、次のように宣言する。
dp[ i ][ K ][ m ] := i 番目までの素材から、 初期の魔力と X との差を K で割ったあまりが m となるように K 個選んだ時の 初期の魔力の最大値。

すると、m が 0 であるような選び方の初期値は、時間が経てばいつか魔力が X になる。
よって、 m が 0 であるような選び方の初期値から、最短時間を計算できる。



壊れた電車


f( x ) := x 分以内に点検ができるか
という判定を考えます。

左の整備士から順に、
    x 分以内に、点検していない車両に当たるまで左に移動し、左側に点検していない車両がなくなれば進めるだけ右に進む。
という操作を考えます。
この操作ののちに、点検されていない車両がなければ x 分以内に点検可能と判定できます。

すると、左から i + 1 番目の整備士は、左から i 番目の整備士よりも左の車両を見る必要がなくなり、これは直感的にも最適な点検方法です(右を見る分には、余った時間で好きなだけ見ても良いし、整備士がいる場所にあたれば、そこで打ち切っても良い)。

すると、関数 f の計算が O( M ) で可能なので、二分探索ができます。

仮に、数列 a が全て互いに異なる数なら、答えは二項係数で |a| Combination k である(|a| 個から k 個選ぶ方法の総数)。
今回は鳩の巣原理で重複する要素のくみがちょうど 1 組だけが a に含まれる。

その重複する 2 つの要素を 左から al , ar とすると、|a| Combination k は以下の場合を重複して数えてしまうことになる。
・al よりも左の要素と、ar よりも右の要素から k-1 個選び、al , ar 農地どちらかを選ぶ。
al , ar のどちらを選んでも、部分列は同じになるので、この選び方で得られる部分列たちは、ちょうど 2 回 選ばれていることになる。

よって、答えは |a| Combination k から 上記の重複する場合の総数を引いたものが答えとなる。( al より左 + ar より右 から k 個選ぶ総数 を引く )



おやつ


dp[i][x] := 値段が高い順に i 番目まで見て、x 円で買えるおやつの満足度の和の max
とする。これは通常のナップサックDPです。

しかしこれだけでは不十分で、任意のおやつを抜いた場合に、P 円以下になれば良いという条件を考えないといけません。

おやつの値段が高い順に見ているので、i 番目のおやつを見ている時点で dp[i] の値に寄与しているおやつは、i 番目のおやつよりも値段が高いことがわかります。
つまり、i 番目のおやつまで見た時、どのおやつを抜いても i 番目のおやつを抜いた場合の値段以下の値段になります。
よって、i 番目のおやつまで見た時、 P + i 番目のおやつの値段 までならおやつを買って良いとわかります(任意のおやつを抜くことで、P 円以下になるので)。

よって、通常通りナップサックDPを行いながら、各おやつを見るタイミングで 満足度の max を更新していけば良いです。

・同じ登校グループに属する人たちの属するクラスの情報を保存する必要がある。

3N Numbers


元の数列を前半後半の二つに分けることを考える(それぞれ、最低サイズが N 以上)。

前半の数列を F 、後半の数列を B とすると、
スコアは "" F の大きい方から N 個の和 - B の小さい方から N 個の和 ""
となり、F , B それぞれにおいて選ばれなかった要素は、元の数列から取り除かれたものとみなすことができる。

よって、仕切りの位を x とすると、x 以前を F , x より後を B とすると、
x が小さい方から見ていくことで、F , B を O( logN ) (平衡二分木などを使うので) で更新可能で、大きい方、小さい方から N 個のの総和も O( logN ) で計算可能。

負辺があるので、ベルマンフォード法を使いましょう。
スコアを最大化したいので、c に -1 をかけることで最小化問題に変換します。

また、スコアがいくらでも大きくなる(頂点 1 から到達可能で 頂点 N に到達可能な負閉路が存在する) → ベルマンフォード法によって解が収束しない。

Factorization


M を素因数分解すると、各素因数がいくつあるかがわかる。

数列 a の各要素は、1 に M のいくつかの素因数をかけたものである(重複あり)。
逆に、M の各素因数に対して、a のどの要素にかけるかを考えたら良い。

各素因数について、個数が x 個であるとしたら、求める場合の数は N 個の区別する箱に x 個の区別しないボールを入れる場合の数と同じである。



ABC311 - G

X 以上の要素だけ残したグリッドを考える。

X 未満の要素がある位置を穴として考えて、穴を含まないような長方形だけを見ると良い。
これは、上から i 行目までを見ると、i 行目を底辺とするような長方形を i 行目を底とするヒストグラムから長方形を取り出す問題と捉えることができます。

具体的には i 行目から上方向に、穴に当たるまでヒストグラムを伸ばすことを考えます。
i 行目 j 列目のヒストグラムの高さ : H_j がボトルネックであるような長方形を取り出すことを考えます。
これは、高さ H_j の長方形を j 列目から左右に伸ばせるだけ伸ばす問題になり、これは蟻本に乗っている問題と同じ解法で解くことができます。

要素が X 以上のグリッドを考えていますので、長方形内の要素の和 × X が評価値になります。
これを、全ての X について計算しても間に合わせることができます。
