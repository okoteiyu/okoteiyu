ポテンシャル(ABC253-F)

                        
初期値が 0 の N 行 M 列の行列に対して Q 個のクエリ(以下の 3 種類)を順に処理せよ
                        
・1 L R X : L 列目から R 列目の全ての要素に +X 加算する
                        
・2 i X : i 行目の要素全てを X に更新する
                        
・3 i j : i 行目の j 列目の値を出力する
                        
 N , M , Q ≦ 200000
行と列を独立に考えたいので、クエリ 1 と クエリ 2 を独立に考えます。
\(t\) 番目のクエリを処理する時刻を 時刻 \(t\) と呼ぶことにします。

ある時刻のある列にはポテンシャルが存在するものとして考えます(初期値は 0)。
クエリ 1 を L 列目から R 列までのポテンシャルに +X 加算する と言い換えます。
以下を定義します。
  • \({A^t}_{i,j}\) := 時刻 \(t\) における \(i\) 行目の \(j\) 列目の値
  • \({p^t}_j\) := 時刻 \(t\) における \(j\) 列目の要素が持つポテンシャル
  • \({f^i}(t)\) := 時刻 \(t\) 以前に最後に \(i\) 行目が一律更新された時刻
\({A^t}_{i,j}\) = (\(X\):最後に行 \(i\) を更新した値) + (\({f^i}(t)\) 以降に列 \(j\) に加算された分)
となります。
\({f^i}(t)\) 以降に列 \(j\) に加算された分というのは、時刻 \(t\) と時刻 \({f^i}(t)\) におけるポテンシャルの差 なので、

\({A^t}_{i,j}\) = \(X + {p^t}_j - {p^{{f^i}(t)}}_j\)

となります。

\({p^t}_j\) の配列サイズは \(O(QN)\) ですが、\(O(Q)\) 個程度の \((時刻,列)\) のペアにのみポテンシャルを参照するので、
\(p\) の中身を全て保持する必要はありません。

あらかじめ参照する \((時刻,列)\) を求めておくことで、
遅延セグメント木 \(pS\) を時系列順に区間更新しながら、各時刻において参照が必要な列のポテンシャルだけ map<pair<int,int> などにメモしておく
ことができます。
\(pS\) を平面走査のように更新するので、サイズの問題も OK です。 
struct qr{
    int type_,l,r,i,j,x;
};
int main(){
    // 全て 1-index とする
    int N , M , Q;
    cin >> N >> M >> Q;
    vector<qr> queries(Q+1);
    vector<vector<int>> refer(Q+1); //[t] := 時刻 t の時点でのポテンシャルを参照するべき列の index
    vector<int> ft(N+1,0); // [i] := 最後に行 i を更新したタイミング (時刻 0 に 0 で更新したものとする)
    for(int t = 1 ; t <= Q ; t++){
        qr& qrt = queries[t];
        cin >> qrt.type_;
        if(qrt.type_ == 1)cin >> qrt.l >> qrt.r >> qrt.x;
        else if(qrt.type_ == 2){
            cin >> qrt.i >> qrt.x;
            // 最後に行 i を更新したタイミング
            ft[qrt.i] = t;
        }else{
            cin >> qrt.i >> qrt.j;
            // ポテンシャルを参照する (時刻,列) をメモ
            refer[t].push_back(qrt.j);
            refer[ft[qrt.i]].push_back(qrt.j);
        }
    }
    map<int,map<int,long long> > p;// [t][j] := 時刻 t における列 j のポテンシャル (0 で初期化されてい都合が良い)
    LazySplayTree<long long> pS;// 区間加算ができるやつ
    REP(i,M+2)pS.push_back(0);
    vector<int> X(N+1,0);// [i] := 最後に行 i を更新した時の値
    vector<int>(N+1,0).swap(ft); // ft を初期化
    for(int t = 1 ; t <= Q ; t++){
        qr& qrt = queries[t];
        if(qrt.type_ == 1)pS.RangeAdd(qrt.l,qrt.r+1,qrt.x);
        else if(qrt.type_ == 2){
            X[qrt.i] = qrt.x;
            ft[qrt.i] = t;
        }else{
            long long Atij = X[qrt.i] + pS[qrt.j].Value - p[ft[qrt.i]][qrt.j];
            cout << Atij << endl;
        }
        // 参照するべき列のポテンシャルを pS から取り出す
        for(int j : refer[t])p[t][j] = pS[j].Value;
    }
    
    return 0;


function composite & reverse

一次関数の列 : \(F_1 , F_2 , \dots , F_n\) があります。
以下のクエリを処理してください。
  • i a b : \(F_i\) を \(F_i(x) = cx+d\) に置き換える
  • l r : \(F_l,F_{l+1} , \dots F_{r-1} , F_r\) を反転する
  • l r x : \(F_r(F_{r-1}(\dots F_{l+1}(F_l(x))))   mod  998244353\) を出力する
この問題は library checker の動的木の問題のコアな部分の要約です。
列の左から関数を入れ子にした合成関数(composite)を求めるのが曲者です。
動的木(Link Cut Tree)の問題なので、一次関数の列 \(F\) は splay 木で持ちます。

splay 木のノードは、そのノード以下の部分木のマージ値を持ちます。
今回の場合は、部分木(に対応する区間)の合成関数の係数をもたせます。
一次関数を合成しても一時関数にしかならないのがミソです。

例えば、\(g(x) = ax + b\) と \(h(x) = px+q\) を合成すると、\(h(g(x)) = (ap)x + (bp+q)\) になります。

以上を踏まえて、splay 木の \(i\) 番目のノードに以下を定義します。
  • c , d : \(F_i(x)\) の係数 (\(F_i(x) = cx + d\))
  • c_composite , d_composite : 部分木の関数を全て左から合成したときの係数
  • c_Rcomposite , d_Rcomposite : 部分木の関数を全て右から合成したときの係数
  • \(C(x)\) := \(F_r(F_{r-1}(\dots F_{l+1}(F_l(x))))\) : (部分木の区間内の関数を左から合成)
  • \(R(x)\) := \(F_l(F_{l+1}(\dots F_{r-1}(F_r(x))))\) : (部分木の区間内の関数を右から合成)
\(C(x) , R(x)\) は定義してあるだけで、メンバとして明示的に持つものではないです。
本題は、\(C(c),R(x)\) の係数 (c_composite など) を求めることです。

\(i\) 番目のノード : \(Y\) の \(C(x)\) は、ノード \(Y\) の左右の子頂点 \(Lef,Rig\) のメンバを用いて以下のように計算できます。
・\(Y.C(x) = Rig.C(F_i(Lef.C(x)))\)
左 → 自分 → 右 順に合成するので、このような順番になります。
逆に、\(R(x)\) は
・\(Y.R(x) = Lef.R(F_i(Rig.R(x)))\)
となります。
\(g(x) = ax + b\) と \(h(x) = px+q\) の合成 が \(h(g(x)) = (ap)x + (bp+q)\) となることを踏まえて、
c_composite , c_Rcomposite などのメンバ変数に合成結果の係数を代入していけば良いです。

また、区間反転は遅延評価で実装します。
\(C(x)\) と \(R(x)\) は合成の順番を逆にしたもの同士なので、区間反転が起こると \(R\) と \(C\) が入れ替わります。
よって、区間反転の遅延を評価する際は、\(R\) と \(C\) をスワップ、つまり
・swap(c_composite , c_Rcomposite)
・swap(d_composite , d_Rcomposite)
すれば良いです。

\(F_i\) を書き換える操作は、通常の splay 木で値を書き換える操作と変わらないので説明不要ですね。

以下、splay 木ライブラリの update & eval 部分です。 
 void update(bool evaluate = true){
    if(isExist(this) == false)return;
    if(evaluate)this->eval();
    if(isExist(this->left) && evaluate)this->left->eval();
    if(isExist(this->right) && evaluate)this->right->eval();
    this->SubTreeSize = 1;
    this->c_composite = this->c;
    this->d_composite = this->d;
    this->c_rcomposite = this->c;
    this->d_rcomposite = this->d;
    
    if(isExist(this->left)){
        this->SubTreeSize += this->left->SubTreeSize;
        this->c_composite = this->left->c_composite*this->c;
        this->d_composite = this->left->d_composite*this->c + this->d;
        this->c_rcomposite = this->c_rcomposite*this->left->c_rcomposite;
        this->d_rcomposite = this->d_rcomposite*this->left->c_rcomposite + this->left->d_rcomposite;
    }
    if(isExist(this->right)){
        this->SubTreeSize += this->right->SubTreeSize;
        this->d_composite = this->right->d_composite + this->right->c_composite*this->d_composite;
        this->c_composite = this->c_composite*this->right->c_composite;
        this->d_rcomposite = this->c_rcomposite*this->right->d_rcomposite + this->d_rcomposite;
        this->c_rcomposite = this->c_rcomposite*this->right->c_rcomposite;
    }
    
    return;
}
void eval(){
    if(this->reverse_flag_lazy){
        swap(this->left , this->right);
        // 反転が必要
        swap(this->d_rcomposite,this->d_composite);
        swap(this->c_rcomposite,this->c_composite);
        //下に伝播
        if(isExist(this->left))this->left->set_lazyReverse();
        if(isExist(this->right))this->right->set_lazyReverse();
        this->reverse_flag_lazy = false;
    }

区間最小値(最大値)カウント

                        
区間内要素のうち、最小値であるものの個数を、平衡二分木などに乗せる
MinCount 的な名前の要素をノードに持たせると良い。
以下は通常のスプレー木の Node クラスの update と eval の部分。 
// Nodeの持っている情報をupdateする
void update(){
    this->eval();
    this->SubTreeSize = 1;
    this->Min = this->Value;
    this->MinCount = 1;
    if(bool(this->left)){
        this->left->eval();
        this->SubTreeSize += this->left->SubTreeSize;
        if(this->Min == this->left->Min)this->MinCount += this->left->MinCount;
        else if(this->left->Min < this->Min){
            this->Min = this->left->Min;
            this->MinCount = this->left->MinCount;
        }
    }
    if(bool(this->right)){
        this->right->eval();
        this->SubTreeSize += this->right->SubTreeSize;
        if(this->Min == this->right->Min)this->MinCount += this->right->MinCount;
        else if(this->right->Min < this->Min){
            this->Min = this->right->Min;
            this->MinCount = this->right->MinCount;
        }
    }
    return;
}
void eval(){
    // 区間代入
    if(this->lazy_update.first){
        this->Value = this->lazy_update.second;
        this->MinCount = this->SubTreeSize;// 全ての要素が一律 Min となる
        this->Min = this->lazy_update.second;
        if(bool(this->left))this->left->set_lazyUpdate(this->lazy_update.second);
        if(bool(this->right))this->right->set_lazyUpdate(this->lazy_update.second);
        this->lazy_update.first = false;
    }
    // 区間アフィン変換 (MinCountに影響なし)
    if(this->lazy_affine.first){
        this->Value = this->lazy_affine.second.first * this->Value + this->lazy_affine.second.second;
        this->Min = this->lazy_affine.second.first * this->Min + this->lazy_affine.second.second;
        if(bool(this->left))this->left->set_lazyAffine(this->lazy_affine.second.first,this->lazy_affine.second.second);
        if(bool(this->right))this->right->set_lazyAffine(this->lazy_affine.second.first,this->lazy_affine.second.second);
        this->lazy_affine.first = false;
    }
