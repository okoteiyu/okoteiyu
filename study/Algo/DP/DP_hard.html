EDPC - Wを解こう

インライン DP

問題 (EDPC-W)

\(M\) 個の区間が与えられるので、あなたは 0 と 1 のみからなる長さ \(N\) の 文字列に対して、以下のようにスコアを計算します。

  • \(i\) 番目の区間を \([ li , ri ]\) としたとき、文字列の \(li\) 文字目から \(ri\) 文字目までに 1 が存在するならばスコアに \(a_i\) 加算する。
文字列を自由に選べるとき、スコアの最大値を答えよ。
制約
  • \(1 \leq N,M \leq 200000\)
  • \(1 \leq li,ri \leq N\)
  • \(-{10}^9 \leq a_i \leq {10}^9\)

解法

以下の DP テーブルを考えます。

  • \(\mathrm{DP}(i,j) :=\) 長さ \(i\) の 文字列で、最後に 1 が現れる位置が \(j\) である文字列に対して、右端が \(i\) 以下の区間のスコアの和の最大値
  • \(\mathrm{DP}(i,i) = \mathrm{max}_x(\mathrm{DP}(i-1,x))+ \sum{(右端が   i   の区間のスコア)}\)
  • \(\mathrm{DP}(i,j) = \mathrm{DP}(i-1,j) + \sum{(右端が   i   で、左端が   j   以下の区間のスコア)}\)
この DP テーブルは \(O(N^3)\) 時間で計算可能です。

ここから計算量を落としていきましょう。\(M\) 個の区間を、右端が小さい順にソートして 1 つづつ見ながら DP テーブルを計算します。

区間 \([L,i] \) を見ている場合を考えます。

まず、DP テーブルの一つ目の添字は明示的に持つ必要はないので DP テーブルをコピーして使うことにしましょう。このコピーも明示的に行うものではなく、以降 DP テーブルの 1 つ目の添字は省略して表記します。

区間 \([L,i] \) を見ているということは、右端が \(i\) 未満の DP の計算は既に完了しており、現在は右端がちょうど \(i\) の計算を行なっています。

\(\mathrm{DP}(i)\) = 「最後に 1 が登場する位置が \(i\) の場合の最大スコア」 の計算は区間 max ができれば良いので、DP テーブルをセグ木で扱い、max をとって \([L,i] \) のスコアを足します。

「最後に 1 が登場する位置が \(i\) 未満の場合の最大スコア」の計算は、最後に 1 が現れる場所が \(L\) 以降である必要性から、\(L\) 以上 \(i\) 未満の \(x\) に対して \(\mathrm{DP}(x)\) に区間 \([L,i]\) のスコアを一律に加算することで全ての \(j<i\) に対して \(\mathrm{DP}(j)\) を計算できています。

二次元の DP テーブルを一次元に落とし、二次元のテーブルを縦(または横) 向きに走査していくような DP を インライン DP と言います。

問題 (ABC353-G)

\(1\) 移動するのに \(C\) 円かかる数直線上にいるとします。各 \( 1 \leq i \leq M\) に対して時刻 \(i\) に座標 \(T_i\) にいると \(P_i\) 円得られるとき、差し引き収益は最大でいくらになりますか。ただし、座標の移動に時間はかからないものとします。

制約
  • \(1 \leq M \leq 200000\)
  • \(1 \leq T_i \leq 200000\)
  • \(1 \leq P_i \leq {10}^{13}\)
  • \(1 \leq C \leq {10}^9\)

以下の DP テーブルを考えます。

  • \(\mathrm{DP}(i,x) := \) 時刻 \(i\) に座標 \(x\) にいる際の最大収益
これもインライン化して 1 つ目の添字を省略して考えます。時刻が早い順にイベントを調べていきます。

時刻 \(i\) の DP テーブルを考えると、時刻 \(i\) の時点で \(T_i\) 以外の座標に移動する意味はないので、\(\mathrm{DP}(T_i) \) のみ新たに計算して、それ以外の地点は時刻 \(i-1\) の結果を引き継げば OK です。

時刻 \(i-1\) に地点 \(x\) におり、時刻 \(i\) に地点 \(T_i\) に移動するという移動ルートのうち、収益が最大のものを選ぶという遷移により、\(\mathrm{DP}(T_i) = P_i + \mathrm{max}_x(\mathrm{DP}(x) - C\cdot |T_i-x| ) \) となります。

\( \mathrm{max}_x(\mathrm{DP}(x) - C\cdot |T_i-x| ) \) の部分、つまり \(T_i\) から離れるだけコストが \(C\) かかるということを、公差が \(C\) の等差数列を区間に加算する操作で表現できます。

  1. \(0 \leq x < T_i\) である区間に対して、初項 \(-C \cdot T_i\) , 公差 \(C\) の等差数列を加算する。
  2. \(T_i < x \) である区間に対して、初項 \(0\) , 公差 \(-C\) の等差数列を加算する。
以上の操作を行った DP テーブルにおける max + \(P_i\) が新たな \(\mathrm{DP}(T_i)\) です。\(\mathrm{DP}(T_i)\) を計算した後は、上記の 2 つの操作を取り消します。これは符号が逆の等差数列を加算すれば良いです。

ところで、区間等差数列加算/区間 max をサポートするデータ構造は難しいです。そこで、区間等差数列加算は線分追加に言い換えるというテンプレを考えます。

Li-Chao Tree を使い、地点を x 座標、収益を y 座標に対応づけます。 すると地点 \(T_i\) から次の地点への移動のコストを織り込んだコストを二次元座標上の線分で表現することができます。

地点 \(T_i\) の収益 (\(F\) とする) を計算したとき、DP テーブルに \(F\) を書き込む代わりに、Li-Chao Tree が表す二次元座標系に 「\(x = T_i\) における y 座標が \(F\) , \(x = T_i\) から 1 離れるごとに \(C\) 減少する直線」を左右の方向合わせて 2 本追加します。離れるごとに \(C\) 減少というのが、 次の地点への移動コストを表現しています。

すると現時刻に地点 \(t\) にいる場合の収益を、「\( x = t\) をカバーする線分の \(x = t\) における y 座標の最大値」と表現でき (これは Li-Chao Tree で計算できます)、いわゆる配る DP のようなアルゴリズムで、インライン DP することができました。 

int main(){
    ll n , c , m;
    cin >> n >> c >> m;
    LiChaoTree<ll,ll> dp(0,n+2,NINF,false);
    dp.add_segment(1,n+2,-c,c);
    REP(i,m){
        ll t , p;cin >> t >> p;
        ll mx = dp.f(t);
        dp.write_segment(t , p + mx , 0, (p + mx) - c*t);
        dp.write_segment(t , p + mx, n+1 , (p + mx) - c*(n+1 - t));
    }
    ll ans = 0;
    REP(i,n)ans = max(dp.f(i+1),ans);
    cout << ans << endl;
    return 0;
