bitDP

概要

非負整数の集合を状態として持つ DP です。

非負整数整数の集合 \(S\) があるとします。\(S\) に含まれる整数を \(1\) として、集合をビットマスクします。例えば集合 \({0,1,3}\) は整数 \(1011_{(2)} = 11\) です。\(N\) 以下の非負整数の集合の組み合わせが \(2^{N+1}\) 個なので、\(N\) が小さい場合のアルゴリズムであることが多いです。例えば、\(N\) が小さい場合の巡回セールスマン問題は以下の DP でかけます。

  • \(\mathrm{DP}(s,v) := \) これまで訪れた頂点番号の集合のビットマスクが \(s\) で、最後に訪れた頂点の番号が \(v\) であるとき、それまでの移動距離としてあり得る最小値。

部分集合 Bit DP

整数の集合 (ビットマスク) \(S\) の部分集合を表すビットマスク \(T\) を、以下の for 文で列挙することができます。 

for(int T = S ; T > 0 ; T = (T-1)&S){
    // if(T == S)continue; // 自分自身を弾く場合
    // if(T == 0)continue; // 空集合を弾く場合
}

なお、\(N\) 以下の非負整数からなる全ての集合について部分集合を調べる処理を行うと、イテレーション回数は \(O(3^{N+1})\) になります。


部分集合包除と高速ゼータ変換

非負整数集合をビットマスクした整数 \(S\) に対して何らかの値を計算する関数 \(f(S)\) に対して、演算の積 \(\oplus\) (\(\sum\) や \(\Pi\) のようなもの) を用いて \(\mathrm{dp}[S] = \oplus_{t\in S}{f(t)}\) を計算します。想像しやすいようにここでは \(\oplus = \Pi\) としましょう。

\(f\) の結果は \(\mathrm{dp}\) に集約されるので、その集約パターンをいい感じにすると、\(\mathrm{dp}\) の値を \(\mathrm{dp}\) の計算に使用することができます。 

for(int S = 0 ; S < (1<<20) ; S++)dp[S] = f(S);
for(int b = 0 ; b < 20 ; b++){
    for(int S = 0 ; S < (1<<20) ; S++){
        if(S & (1 << b))dp[S] *= dp[S & ~(1 << b)];
    }
}

意味としては、下から \(b\) ビットの自由度があり、他は \(S\) と同じ状態からの遷移を、インライン DP で \(b\) の昇順に計算するものですが、基本的にテンプレとして覚えれば良いかなとの気持ちです。
