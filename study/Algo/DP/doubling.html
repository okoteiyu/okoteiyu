ダブリング

ダブリングとはなんですか -> \(2^{k-1}\) 個先のさらに \(2^{k-1}\) 個先は \(2^k\) 個先という事実、これにつきます。ダブリングの目的は、ある状態から \(x\) 回遷移した時、どの状態になっているかを求めることです。

ダブリングすることができる対象は、Functional Graph (各頂点から、高々 \(1\) 本以下の辺が出る有向グラフ) です。


例えば、このグラフで頂点 \(1\) から辺に沿って \(4\) 回移動した結果は一意に \(11\) と定まります。


これを効率的に計算しますが、その前に愚直な方法として以下のような DP を考えてみましょう。

  • \(\mathrm{DP}(s,x) := \) 状態 (頂点) \(s\) から \(x\) 回遷移 (移動) した結果
これは各頂点から愚直に \(x\) 回移動を行うことで簡単に求まりますし、それよりも良い計算量でこの DP テーブルを埋めることはできません。\(x\) の上限は \(10^{18}\) などもあり得るので、これは良くないです。

ところで、\(x = a + b\) と書けるとき、\(\mathrm{DP}(s,x) = \mathrm{DP}(\mathrm{DP}(s,a),b)\)、つまり \(\mathrm{DP}(s,x) \) は \(s\) から \(a\) 移動してさらに \(b\) 移動したものですよね。

mod library の記事の modpow で紹介したように、\(x\) を二進数表示してビットが \(1\) の部分 \(b_0,b_1,b_2,\dots\) に対して \(s\) から順番に \(2^{b_i}\) ずつ移動すると、\(x = \sum{2^{b_i}}\) なので \(s\) から \(x\) 移動した地点に到達しますね。

よって、用意するべき DP テーブルは以下でした。

  • \(\mathrm{DP}(s,b) := \) 状態 (頂点) \(s\) から \(2^b\) 回遷移 (移動) した結果
  • \(\mathrm{DP}(s,b) = \mathrm{DP}(\mathrm{DP}(s,b-1),b-1) \) を \(b\) が小さい順に計算できます。

(modpow (高速累乗) は DP テーブル (遷移の有向グラフ) を陽に持たないダブリングと言えます。) 



#include<iostream>
#include<vector>
const long long INF = 1e18;
using namespace std;
/*
    A = {A_1,A_2, ... , A_N}があります。
    i は A_iに一方通行です。
    最初1にいるとき、K(K<10^18)回道を移動した後あなたはどこにいますか。
*/
int main(){
    int N ;long long K ;
    cin >> N >> K;
    vector<vector<int>> dp(65 , vector<int>(N+1));
    vector<int> A(N);
    for(int i = 0; i<N;i++)cin >> A[i];
    for(int i = 1 ; i <= N ; i++){
        dp[0][i] = A[i-1];
    }
    for(int k = 1 ; k < 65 ; k++){
        for(int x = 1 ; x<=N ; x++){
            dp[k][x] = dp[k-1][dp[k-1][x]];//2^個先は、2^(k-1)個先の2^(k-1)個先
        }
    }
    int now = 1;
    for(int bit = 0 ; bit < 64 ; bit++){
        if((K & (1ll<<bit))!=0){
            now = dp[bit][now];
        }
    }
    cout << now << endl;
    return 0;
