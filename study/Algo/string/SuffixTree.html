Overview

接尾辞木(Suffix Tree) という最強データ構造を構築します。ある文字列 \(S\) の部分文字列の種類数は \(O(|S|^2)\) 個ですが、Suffix Tree は \(S\) の区間の参照をうまく木構造に乗せることで、\(O(|S|)\) words の空間計算量で実質的に \(S\) の全ての種類の部分文字列に相当するデータを列挙できます。


構築の計算量も \(O(|S|\log{\sigma})\) 時間 (\({\sigma} :=\) アルファベットサイズ) と高速なため、\(S\) の部分文字列に関する計算を行う問題ではまず Suffix Tree を考えることが有効です。実際 ABC268-Ex (2560 diff) や Library Checker の Longest Common Substring は、考察が難しい部分を Suffix Tree で表現することができ、いとも簡単に解けてしまいます。

後に述べますが、Suffix Array と LCP Array さえあれば Suffix Tree の構築自体はそこまで難しい話ではないです。本記事では、簡単な Suffix Tree 構築アルゴリズムと、Suffix Link と push_back を用いた Suffix Tree 構築アルゴリズムを紹介します。

Suffix Tree の構築に関しては、\(2\) 章と \(3\) 章で異なる構築方法を示しますが、基本的に \(3\) 章で示すものの方が強いです。ただしその分若干実装が大変です。

前準備

必要なデータ

Suffix Tree は隣接リストで表現可能な木構造ですが、Trie と同様に辺ラベルから隣接頂点を求める索引を持つ必要があります。また構築中に使用したり、様々な問題を解くために使用するデータもあるので、各頂点には以下のようなデータを持たせる必要があります。以下のメンバを持つオブジェクトを Node クラスとして定義します。

  • parent := 親頂点
  • child[x] := この頂点から出る辺のうち、辺ラベルの先頭が x の辺の先にある頂点
  • label.first := この頂点に入る辺のラベルの参照 (半開区間の左端)
  • label.second := この頂点に入る辺のラベルの参照 (半開区間の右端)
  • path_label_length := 根からこの頂点までの辺ラベルの文字数の和
  • suffix_id := この頂点が対応する接尾辞が \(S\) の何文字目から始まるか。接尾辞に対応しない場合は \(-1\) とする。
  • prod_data := この頂点以降の部分木に関する集約データ
  • prod_op := この頂点以降の部分木に作用する遅延評価

また、Suffix Tree 上の位置を示すデータを以下の形式で定義します。Suffix Tree では辺の途中の位置を示す必要があるので、以下のデータで定義します。

  • \((u, v, p) := \) 頂点 \(u\) と頂点 \(v\) をつなぐ辺を、始点から辺ラベルに沿って \(p\) 文字移動した場所
  • 頂点 \(x\) を示す位置データは \((x, \phi, 0)\) とする。

これらのデータを定義することで、Suffix Tree の表現と、Suffix Tree 上の操作を定義することができるようになります。のちに説明しますが、Suffix Tree 上の位置データは文字列 \(S\) の \(1\) 種類の部分文字列と一対一対応しています。


Suffix Array と LCP Array

静的な Suffix Tree 構築では、Suffix Array と LCP Array が必要になります。これらの構築自体も難しい問題ですが、これらは一般的によく知られているため、ここでは詳細は割愛して定義のみ説明することにします。

文字列 \(S\) に対する Suffix Array SA を以下で定義します。

  • SA[i] := \(S\) の空文字列を含む接尾辞のうち、辞書順で \(i\) 番目に小さいものの、\(S\) における開始 index
  • 末尾の空文字列 \(\phi\) も接尾辞に含むものとし、空文字列は他のどの文字列よりも辞書順で小さいものとする。

文字列 \(S\) に対する LCP Array LCP を以下で定義します。

  • LCP[0] = 0
  • LCP[i] := SA[i-1] と SA[i] が表す \(2\) つの接尾辞の最長共通接頭辞(先頭何文字が一致しているか)

これらはどちらも線形時間で構築することができます。また、push_back によるオンライン構築ではこれらの列は不要であり、構築した Suffix Tree から逆にこれらを構築することもできます。


構築方法

まず、\(S\) の接尾辞を \(1\) つずつ木に格納する実装を考えます。

何も格納されていない初期状態では、根頂点のみが存在します。接尾辞の格納では、Trie と同様に既存の辺を共有する位置まで移動し、共有できない部分があれば分岐させて新たな葉を追加します。


この処理では必要なデータの項で用意したものを活用します。例えば、上の図で ookoook を格納する際は、位置 \((0,5,1)\) に頂点 \(3\) を作成し、新たにできた頂点 \(3\) の影響を受ける部分のデータを再計算します。

しかし、このままでは共有する位置まで移動するのに毎回 \(O(|S|)\) 時間かかるため、全体で \(O(|S|^2)\) 時間かかってしまいます。結論から言うと、接尾辞の辞書順に接尾辞を格納することで、木上の移動を工夫することができます。


上図は接尾辞の辞書順に接尾辞を格納する様子ですが、これだけを見ると何も工夫がないように見えます。しかし、実はこの順で格納する場合、共有する位置は一つ前に格納した接尾辞との LCP に対応します。\(S\) のある接尾辞 \(\mathrm{sf}\) は、\(S\) の Suffix Tree において他の接尾辞と辺を共有しますが、辞書順の定義より、この時共有する辺の長さは Suffix Array において \(\mathrm{sf}\) と隣り合う接尾辞との LCP の長さに一致するからです。

なお、接尾辞の辞書順は Suffix Array で定義されており、計算に必要な LCP は LCP Array で計算されています。また Suffix Tree 構築中は、最後に格納した接尾辞の位置(頂点)を記録しておく必要があります。


\(i\) 回目の格納操作では LCP[i] まで移動する必要があります。まず、最後に格納した SA[i-1] に対応する頂点から、親頂点への移動を繰り返すことで、対応する文字列の長さが LCP[i] の長さ以下である頂点 まで移動します。上図の例では、最後に格納した頂点 \(2\) から、koo 以前の頂点である頂点 \(1\) まで移動します。

移動後の頂点を \(x\) とすると、\(x\) に対応する文字列の長さを参照することで LCP[i] の位置までの残りの文字列を復元することができます。この時、LCP[i] に対応する位置は頂点 \(x\) から出る辺上、もしくは頂点 \(x\) そのものであり、LCP[i] までの残りの文字列の情報を参照することで LCP[i] の位置を \(O(1)\) 時間で復元することができます。


親頂点への移動も LCP[i] までの移動も、辺上の位置を \(1\) 文字ずつ移動するのではなく、目的の位置まで一気にジャンプします。これは位置データ \((u,v,p)\) の定義から \(O(1)\) 時間で実行可能であることがわかります。

push_back でオンライン構築する方法 (Suffix Array/LCP Array 不要)

この方法では、文字列 \(S\) に文字を push_back しながら、その影響を \(S\) の Suffix Tree にリアルタイムに反映します。この時、リアルタイム構築中の Suffix Tree が、前章の冒頭で示したものになります。すなわち、構築中の Suffix Tree においては、対応する頂点を持たない接尾辞が存在する可能性があります。しかし構築中の Suffix Tree であっても部分文字列の全列挙は達成されており、部分文字列検索など、補助的なデータを必要としない操作であれば問題なく実行できます。また、これ以上 push_back されないことが確定したタイミングで、Suffix Tree を変形して、各接尾辞に対して対応する頂点を生成することもできます。

改めてですが、Suffix Tree の頂点は何らかの文字列に対応します。先述の構築方法では特筆しませんでしたが、今回の構築方法ではより形式的にデータを表現する機会が多いので、改めて以下を定義します。

  • \(\mathrm{label}(x) := \) 頂点あるいは位置データ \(x\) に対応づけられた文字列
  • \(\mathrm{walk}(x,\mathrm{str}) := \) 頂点 \(x\) から、文字列 \(\mathrm{str}\) に沿って移動した位置あるいは頂点

上記 \(2\) つは、Suffix Tree に定義したデータによって(論理的に)計算可能です。そして、この方法では次の節で説明する Suffix Link が必須になります。Suffix Link は、2.2.1 項で定義した Node クラスのメンバに追加で定義します。


Suffix Link

頂点 \(x\) の Suffix Link として、\(\mathrm{sLink}(x)\) を以下で定義します。

  • \(\mathrm{sLink}(x) := \) \(\mathrm{label}(x)\) の先頭の文字を取り除いた文字列に対応する頂点

ただし、ここでは空文字列 \(\phi\) も接尾辞に含まれるものとして定義します。すると、全ての文字列は空文字列 \(\phi\) を接尾辞に持つため、全ての頂点からは Suffix Link を辿って根に到達可能であることがわかります。また、Suffix Tree の構築中の手続きの共通化のため、根からは根にリンクしているものとして扱います。


構築の手続き

この構築方法では「\(S\) の接尾辞のうち、部分文字列として \(2\) 回以上登場する最長接尾辞」に対応する位置を保存しながら手続きを行います。この位置データを \(F\) として定義します。これから示す手順は全て \(F\) をもとにして行います。

はじめ、\(S\) は空文字列であり、Suffix Tree は空文字列に対応する根頂点のみからなります。また前節で述べた通り、手続きの共通化のために根の Suffix Link は根にリンクしているものとして扱います。


繰り返しますが、この構築では \(S.\mathrm{push\_back}(x)\) を行った結果を Suffix Tree に反映させる方針で構築を行います。\(S.\mathrm{push\_back}(x)\) の影響の反映の形式的な手続きを以下で説明します。形式的なので、飛ばして構築の様子を見ながら参照しても大丈夫です。

  1. Suffix Tree の葉ノードに入る全ての辺の辺ラベルの末尾に \(x\) を追加する。
  2. \(F\) の位置から文字 \(x\) の方向に移動できるなら、\(F\) をその方向に \(1\) 文字分移動して影響の反映を終了する。
  3. \(F\) の位置から文字 \(x\) の方向に移動できない場合。
    1. \(F\) の位置を分岐点として、\(\mathrm{label}(F)+x\) に対応する葉を生成する。この時、\(F\) の位置に頂点がなければ \(F\) に対応する分岐点も生成する。
      • これより前に最後に作った葉から、\(\mathrm{label}(F)+x\) に対応する葉に Suffix Link をリンクする。
    2. \(F\) を \(\mathrm{label}(F)+x\) に対応する葉に更新し、以下を繰り返す。
      1. \(F\) の先頭の文字を削除したものを、新たに \(F\) とする。
      2. \(F\) に対応する位置が Suffix Tree に存在しないなら、\(F\) に対応する葉ノードを生成し、\(F\) をその葉ノードに移動する。必要に応じて分岐点も生成する。
      3. \(F\) に対応する位置が Suffix Tree に存在するなら、\(F\) をその位置に移動して影響の反映を終了する。
      • 移動前の \(F\) に枝分かれする分岐点から、移動後の \(F\) に枝分かれする分岐点に Suffix Link をリンクする。
      • \(\mathrm{label}(F)+x\) に対応する葉が新たに生成されたなら、これより前に最後に作った葉から Suffix Link をリンクする。
補足説明

Suffix Tree に格納される接尾辞は、Suffix Tree の内部の(葉でない)位置に存在することもあります。\(F\) の定義に注意すると、\(S\) の接尾辞のうち \(\mathrm{label}(F)\) 以降の接尾辞は全て Suffix Tree の内部の(葉でない)位置に存在することがわかります。逆に、\(\mathrm{label}(F)\) よりも長い接尾辞は全て対応する葉ノードを持ちます。

葉ノードに対する影響は、上記に手続き \(1\) の通り、辺ラベルを変更するだけで済みます。これは Suffix Tree に格納されている \(\mathrm{label}(F)\) よりも長い接尾辞への影響を反映したことを意味します。よって残る問題は \(\mathrm{label}(F)\) 以降の接尾辞に対する影響の反映のみになります。

\(S.\mathrm{push\_back}(x)\) の影響で新たな葉ノードができる時、その葉ノードは \(\mathrm{label}(F)\) のいずれか接尾辞に \(x\) を push_back したものに対応します。\(\mathrm{label}(F)\) の接尾辞に push_back(x) したものが葉ノードになるかどうかは、\(\mathrm{label}(F)\) の接尾辞の長さに対して単調性があります。上記の \(2\) と \(3\) の手続きによって、\(\mathrm{label}(F)+x\) の接尾辞を長い順に確認しているのです。\(\mathrm{label}(F)+x\) が葉でない位置にあるなら、それは \(F\) の条件を満たすので、それを新たに \(F\) として手続きを終了します。


構築の様子

はじめ、\(S\) は空文字列であり、Suffix Tree には根ノードのみが存在します。この時 \(F\) は空文字列に対応しており、根ノードすなわち位置データ \((0, \phi, 0)\) を表します。


次に、\(S\) に push_back(o) を行います。\(\mathrm{label}(F)+x\) は Suffix Tree に存在しないので、手続きに従い \(\mathrm{label}(F)+x\) に対応する葉を作成し、\(F\) をそこに移動します。


その後手続きに従い、\(F\) を \(\mathrm{label}(F)\) の先頭の文字を削除したものに対応する位置に移動します。今回は根ノードがそれに対応します。


移動方法の説明

ここで、\(\mathrm{label}(F)\) の先頭の文字を削除したものに対応する位置に移動する方法を説明します。ここでは、\(\mathrm{label}(F)\) の先頭の文字を削除したものに対応する位置を \(\mathrm{next}(F)\) とし、\(F\) の位置データを \((u, v, p)\) とします。

辺の始点のデータは構築済みであるため、辺の始点 \(u\) からは Suffix Link を移動することができます。ここで、\(u\) から \(F\) への移動経路を \(w\) としたとき、\(\mathrm{next}(F)\) は、\(\mathrm{sLink}(u)\) から \(w\) だけ移動した位置になります。これは \(\mathrm{walk}(\mathrm{sLink}(u),w)\) で定義されており、Suffix Tree に定義したデータから計算可能です。

ただし、\(u\) が根の場合は例外です。共通化のため、\(\mathrm{sLink}(根)\) は根にリンクされているため、先頭の文字を削除するという部分を、\(w\) の先頭を削除するという処理で表現する必要があります。よってこのとき、\(\mathrm{next}(F)\) は \(\mathrm{sLink}(u)\) から「経路 \(w\) の先頭要素だけ無視した移動経路」だけ移動した位置になります。

\(F\) の定義より新たな葉の位置は、分岐点から \(x\) 一文字分だけ分岐した位置にあるため、\(w\) の長さはちょうど \(1\) になります。このことから、\(\mathrm{next}(F)\) は、\(\mathrm{sLink}(u)\) さえ分かれば後はそこから高々 \(1\) 文字移動するだけで求まります。


移動方法の説明に従うと、現在 \(u = 0,\mathrm{sLink}(u) = 0\) であり、経路 \(w\) は文字列 o で表されます。始点 \(u=0\) は根であるため、\(\mathrm{next}(F)\) は \(\mathrm{sLink}(0)\) から (空文字列) だけ移動した地点になります。

次に、\(S\) に push_back(k) を行います。手続きにある通り、辺 \((0-1)\) の辺ラベルに追記してから、前回同様に \(\mathrm{label}(F)+x\) に対応する葉を作成し \(F\) をそこに移動します。その後、上記と同様に \(\mathrm{next}(F)\) に移動します。

補足説明

ラベルに追記する必要がある辺は \(O(|S|)\) 個存在するので、これらを愚直に更新すると push_back ごとに \(O(|S|)\) 時間かかってしまいます。よって、辺ラベルに対する追記を一括で行うデータ構造を辺ラベルに採用する必要があります。これは全く難しくないですが、今はややこしくなるので後で紹介します。


また、手続きに従いこれよりも前で最後に生成した葉ノードから、今作成した葉ノードに Suffix Link を繋げます。

次に、\(S\) に push_back(o) を行います。前回同様に辺ラベルに追記したのち、今後は \(\mathrm{label}(F)+\)o の位置が Suffix Tree に存在するので、\(F\) をそこに移動して手続きは終了になります。


次に、\(S\) に push_back(o) を行います。今回は葉を作成するための分岐点を先に作成する必要があります。その後は同様に \(\mathrm{label}(F)+x\) に対応する葉を作成し \(F\) をそこに移動してから \(\mathrm{next}(F)\) を確認しますが、\(\mathrm{next}(F)\) を計算するために、今作成した分岐点の Suffix Link を構築する必要があります。

分岐点の Suffix Link の構築

今作成した分岐点の Suffix Link 先は \(\mathrm{next}(F)\) の計算と同じような方法で計算できます。異なる点は、復元する移動経路の長さが \(1\) とは限らないことです。そして結論から言うと、今作成した分岐点の Suffix Link 先には、すでに頂点が存在するか、この直後に作成されることが言えます。詳しくは説明しませんが、これは \(F\) の定義などから説明できます。


分岐点の Suffix Link を構築したのち、\(\mathrm{next}(F)\) を計算します。今回は \(u\) が根ではないので、\(\mathrm{next}(F)\) は \(\mathrm{walk}(\mathrm{sLink}(u),w)\) です。\(\mathrm{walk}\) は、\(\mathrm{label}\) や辺ラベルの長さ、移動経路の長さ、次に移動する方向(文字)を参照しながら計算することができます。

続けて、push_back(k), push_back(o), push_back(o) を行います。これらでは新たな葉ができることはなく処理が進みます。



次に push_back(o) を行います。今回は分岐点を生成してから葉を生成します。


その後、\(\mathrm{next}(F)\) の計算のためにまず分岐点の Suffix Link を作ります。前述の手順で Suffix Link 先の位置を計算したところ、今回はその位置に頂点が存在しなかったため、その位置に新たに頂点を生成します。新たに頂点を生成した場合、\(\mathrm{next}(F)\) に対応する地点は Suffix Tree に存在せず、ここで作成された頂点はその直後に \(\mathrm{next}(F)\) に対応する葉の分岐点として使用されます (このことも \(F\) の定義から説明できます)。


分岐点の Suffix Link (及びリンク先頂点) を生成したのち、\(F\) を \(\mathrm{next}(F)\) に移動させます。今回は \(\mathrm{next}(F)\) が Suffix Tree に存在しなかったため、対応する葉を新たに生成します。


なおこの状態では \(F\) は \(F\) の定義を満たしません。手続きに従い、\(F\) が定義を満たすようになるまで \(\mathrm{next}(F)\) への移動を繰り返す必要があります。ここでも同様に、\(F\) に分岐する頂点の Suffix Link の構築から行う必要があります。

補足説明

今回も \(\mathrm{walk}\) を計算する必要がありますが、今回はこれまでと異なり、\(\mathrm{walk}\) の移動先の復元が複数の辺にまたがっています。このような場合も、\(\mathrm{label}\) や辺ラベルの長さ、移動経路の長さ、次に移動する方向(文字)を参照しながら、移動先の計算を繰り返すことで、\(\mathrm{walk}\) を計算する必要があります。

このように \(\mathrm{walk}\) の計算は定数時間ではありませんが、Suffix Tree 構築全体を通して評価すると線形時間になることが言えます。照明は割愛しますが、\(\mathrm{walk}\) の計算内のイテレーション回数が増える条件を考えると証明できると思います。


その後、同様に \(F\) を \(\mathrm{next}(F)\) に移動します。今回も対応する位置が存在しなかっため、新たに葉を生成してイテレーションを続行します。


同様に \(F\) に分岐する頂点のリンク先を復元し、その後 \(\mathrm{next}(F)\) を探します。今回は \(\mathrm{next}(F)\) が Suffix Tree に存在するので、\(F\) を \(\mathrm{next}(F)\) に移動してイテレーションを終了します。


次に、push_back(k) を行います。今回は \(\mathrm{label}(F)+\)o の位置が Suffix Tree に存在するので、\(F\) をそこに移動して手続きは終了になります。


こうして、\(S = \) okookoook の Suffix Tree ができました。これは冒頭で示したものと一致することがわかります。

辺ラベルのデータ構造

push_back の手続きで、Suffix Tree の既存の辺のラベルを一気に更新したいと説明しました。ここではそれを可能にするデータ構造を紹介します。とはいえ、とても簡単なのであとは消化試合です。

概要で説明した通り、Suffix Tree の辺は文字列 \(S\) の部分文字列の参照である半開区間を持ちます。


push_back で更新される辺ラベルの集合は、手続きより、半開区間の右端が \(S\) の終端であるものと一致します。そこで、\(S\) の終端を表す特殊なシンボルとして $ を採用します。


$ は論理的なシンボルであり、push_back を行った際に自動でその値(実体)が変化します。その結果、$ を右端に持つ辺ラベルは、push_back を行った際に自動的に右端の値(実体)が変化します。


この $ は参照やポインタによって簡単に表現することができます。これにて、Suffix Tree を完璧に構築することができます。


接尾辞木を用いて解ける問題の簡単な説明

Suffix Tree ライブラリをページ最下部のサンプルコード欄に掲載しています。ここでは、Suffix Tree ライブラリを用いて解くことができる問題の例を紹介します。


接尾辞木を用いた符号化

みなさんはローリングハッシュをご存知でしょうか。ローリングハッシュは文字列のハッシュ値を計算する手法で、その強みの一つとして、与えられた文字列の部分文字列に対するハッシュ値の計算が定数時間であることなどがあります。その反面、ハッシュ値への変換は単射でないため、ハッシュ値の一致判定結果が元の文字列の一致判定結果と 100% 一致すると言い切れない問題があります(ハッシュの衝突)。

もし、部分文字列を取りたい文字列群 \(S_1,S_2,\dots,S_N\) が予め分かっているならば、これらをすべて順に連結した文字列の Suffix Tree を構築することで、Suffix Tree 上の位置がこれらの文字列の部分文字列の符号になります。繰り返しですが、Suffix Tree 上の位置データは元の文字列の \(1\) 種類の部分文字列と一対一対応しています。\(S_1+S_2+\dots+S_N\) の部分文字列は、もともと取得する予定の部分文字列を含むため、\(S_1+S_2+\dots+S_N\) が一つの Suffix Tree を共有することで、Suffix Tree 上の位置データを共通の符号として使用できるようになります。

位置データ \((u,v,p)\) は \(3\) つの整数またはポインタのため、この符号のサイズは \(3\) words と小さいです。なお、\(S_1+S_2+\dots+S_N\) の半開区間 \([l,r)\) に対応する位置データは、\(l\) 文字目から始まる接尾辞に対応するノードから、親ノードに関するダブリングテーブルを用いて \(O(\log{\sum{{S_i}}})\) 時間で復元することができます。

この符号はクエリごとの符号化が高速かつ省メモリであることに加えて、ハッシュの衝突が発生しないという利点があります。ただしローリングハッシュとは異なり、文字列の連結結果のハッシュ値の定数時間の計算を行うことはできません。


Suffix Array / LCP Array

根から「辺ラベルの先頭の文字の辞書順が小さい方向が優先の DFS」を行い、道中の終端ノードの suffix_id を順に格納すると Suffix Array ができます。こうしてできた Suffix Array の隣接頂点に対して、それらに対応する Suffix Tree のノードの LCA (Lowest Common Ancestor) を計算することで LCP Array が得られます。


LCA を活用する問題

Suffix Tree における LCA は、\(2\) つの位置データを受け取り、それらの共通祖先の位置データを返します。ここでは、必ずしも LCA は頂点になるわけではなく、辺ラベルの途中の位置なども LCA になり得ます。

Suffix Tree 上の \(2\) つの位置を選ぶことは、元の文字列の部分文字列から \(2\) つ選ぶことと対応します。それらの \(2\) つの LCA を取る操作は、\(2\) つの部分文字列の最長共通接頭辞を取得する操作に等しいです。このことを利用していくつかの有名問題を解くことができます。以下にいくつかの例を紹介します。

  • LCP Array : Suffix Array の隣接要素の LCA を計算する。
  • 全体と接頭辞の LCP (Z-algorithm の計算対象) : 文字列全体と各接尾辞の LCA を計算する。
  • 回文半径 : Manacher と同様に、元の文字列 \(S\) の文字の間に番兵を入れて偶数長の回文半径を計算可能にする。文字列 \(S\) と、文字列 \(S\) を反転した文字列を、どこにも登場しない仕切り文字 \(\overline{c}\) を挟んで連結すると、回文半径は \(S\) における回文の中央から始まる接尾辞と、反転後の \(S\) における同じ回文の中央から始まる接尾辞の LCA から、回文半径を計算することができる。
回文半径のコード例 
void Palindromes(){
    string s;cin >> s;
    string S = "?";
    for(char c : s){
        S += c;
        S += '?';
    }
    string rS = S;
    reverse(rS.begin() , rS.end());

    // [ S , 仕切り , S の逆順 ] を入れる

    T.build(S+"$"+rS);

        decltype(T)::TreePlace n1 = T.search(i+2, S.size()+1);        // 前半の部分文字列 (最後まででも別に良いが、search のデバッグを兼ねる)
        decltype(T)::TreePlace n2 = T.search(T.size()-i-1, T.size()); // 後半の部分文字列

        int r = lca.path_label_length();

            r = (r-1)*2 +1;
        }
        cout << r << " ";
    }  
    cout << endl;


Suffix Link を活用する問題 (Matching Statistics)

有名というわけではありませんが、Suffix Link を用いて \(2\) つの文字列 \(S, T\) に対して以下のデータを計算することができます。

  • \(\mathrm{ms}[i] := T\) の \(i\) 文字目から始まる部分文字列と、\(S\) の任意の部分文字列との共通接頭辞のうち、最長のものの長さ

これは、Aho-Corasick 法と同様に、行き止まりに到達したら Suffix Link を辿って次の接尾辞に移動する手法で計算することができます。具体的には、\(S\) の Suffix Tree の根から \(T\) に沿って移動を行い、移動できた深さから最長共通接頭辞を求めます。はじめは、\(\mathrm{ms}[0]\) の計算から始まります。移動ができない地点に到達したらこれ以上長い共通接頭辞は存在しないことが言えるので、Suffix Link を移動することで \(T\) の次の接尾辞の \(\mathrm{ms}\) の計算に移行します。

コード例 
vector<int> matching_statistics(string s , string t){
    SuffixTree<char> T;
    vector<int> ms(t.size() , 0);
    T.push_back(s); 
    auto now = T.from_root(false); 
    int r = 0;
    for(int i = 0 ; i < t.size() ; i++){
        r = max(i,r);
        while(r < t.size() && T.walkdown(now,t[r]))r++;
        ms[i] = now.path_label_length();
        // 位置データと対応する列が非明示の場合は、引数に渡す
        now.walkSuffixLink(t,i);
    }
    return ms;
}

木上の集約を活用する問題

何度でも言いますが、\(S\) の Suffix Tree は \(S\) のすべての種類の部分文字列を論理的に列挙します。ただし \(|S|\) の部分文字列の種類数は \(O(|S|^2)\) 個存在するため、すべての種類の部分文字列に対して何かを計算したい場合は、\(O(|S|^2)\) 個の部分文字列から集約できる何らかのデータを集約する必要があります。集約の方法は部分木に注目して木 DP を行うなり、辺ラベルの区間データに注目して何らかのデータ構造を作るなり様々です。

例えば、ある部分木に対して「その部分木以降の位置データの種類数」を木 DP で集約しておけば、辞書順で \(x\) 番目の部分文字列を \(O(|S|\log{\sigma})\) 時間で特定することができます。

また部分文字列の文脈とは関係なしに、データの集約によって様々な問題を解くことができます。例えば Overview で紹介した Longest Common Substring の問題は、\(2\) つの文字列 \(S,T\) を、いずれにも登場しない文字 \(\overline{c}\) を挟んで連結した文字列の Suffix Tree において、以下の \(2\) つの DP テーブルを構築することで解くことができます。

  • \(\mathrm{dp}_{S}[x] := \) 頂点 \(x\) 以下の部分木に、文字列 \(S\) 由来の接尾辞に対応する終端ノードが存在するか
  • \(\mathrm{dp}_{T}[x] := \) 頂点 \(x\) 以下の部分木に、文字列 \(T\) 由来の接尾辞に対応する終端ノードが存在するか
Longest Common Substring のコード例 
void LongestCommonSubstring(){
    string s , t;cin >> s >> t;
    SuffixTree<char,false> T,SS;
    T.build(s+"?"+t);// s + ? + t を入れる

    vector<int> dp_s(T.tree_size() , 0);// [x] := 頂点 x の部分木に s の 終端ノード が存在するか
    vector<int> dp_t(T.tree_size() , 0);// [x] := 頂点 x の部分木に t の 終端ノード が存在するか
    auto dfs = T.dfs(); 
    reverse(dfs.begin() , dfs.end());
    for(auto now : dfs){
        if(now->isEndPoint()){
            if(now->suffix_id == s.size())continue;
            if(now->suffix_id < s.size())dp_s[now->id] = 1;
            else dp_t[now->id] = 1;
        }
        if(now->parent != nullptr)dp_s[now->parent->id] += dp_s[now->id];
        if(now->parent != nullptr)dp_t[now->parent->id] += dp_t[now->id];
    }
    string ans = "";
    const decltype(T)::Node* best = T.get_root();
    for(auto now : dfs){
        if(dp_s[now->id] == 0 || dp_t[now->id] == 0)continue;
        if(best->path_label_length() < now->path_label_length())best = now;
    }
    while(best!=nullptr){
        string tmp = "";
        for(int i = best->cleft(); i < best->cright() ; i++)tmp += T[i];
        reverse(tmp.begin(),tmp.end());
        ans += tmp;
        best = best->parent;
    }

    reverse(ans.begin(),ans.end());
    if(ans.size() == 0){
        cout << "0 0 0 0" << endl;
        return;
    }
    // メモリ解放 & 再利用
    T.clear();
    T.build(t);
    SS.build(s);
    // 接尾辞のうち、ans を含むものの開始位置
    decltype(SS)::TreePlace ps = SS.search(ans);
    decltype(T)::TreePlace pt = T.search(ans);
    int a = SS.decode_suffix(ps);
    int c = T.decode_suffix(pt);
    cout << a << " " << a + ans.size() << " " << c << " " << c + ans.size() << endl;


木上の作用を活用する問題

木に関する一般的なテクニックとして、いくつかの箇所に遅延評価を載せておき、DFS の際に木全体に降ろして作用させるというものがあり、Suffix Tree も例に漏れずこのテクニックを使うことができます。Overview で紹介した ABC268-Ex (2560 diff) も、Suffix Tree 上に遅延評価を持たせて最後にまとめて作用させることで、答えを導き出すために必要なデータを構築することができます。

この問題の概要は、\(N\) 個の文字列 \(T_1,T_2,\dots,T_N\) のすべてが文字列 \(S\) の部分文字列にならないように、文字列 \(S\) の文字を * で置き換える際の最小置き換え回数を求めるというものです。突飛ですが、この問題は以下のデータを用意することで解くことができます。

  • \(\mathrm{interval\_right}[i] := i\) 文字目以降、初めて登場する * の位置として許容できる箇所の最大値
  • \(i\) 文字目以降に * が登場する必要がない部分については未定義とする。

すべての \(i\) について、\(T_i\) に対応する箇所に値 \(|T_i|\) を記録しておくことで、DFS の際にこれらの値との min を作用させながら値を集約し、終端ノードに到達したタイミングで集約した値から \(\mathrm{interval\_right}\) の値を計算することができます。

このように構築した \(\mathrm{interval\_right}\) は、未定義である部分以外はすべて * が登場するべき index の制約を表しています。すなわち、\(\mathrm{interval\_right}\) によって定義される未定義でない区間全てについて、それぞれの区間内に少なくとも \(1\) つは * が登場する必要があります。このとき、* に置き換える必要がある文字の最小個数は 区間スケジューリング問題 に帰着することができます。

コード例 

template<typename T>
int RangeSchedulingProblem(vector<pair<T,T>> intervals){
    int res = 0;
    if(intervals.empty())return res;
    sort(intervals.begin(),intervals.end(),[](auto a, auto b){return bool(a.second < b.second);});
    int right = intervals.front().first;
    for(auto x : intervals){
        if(right <= x.first){
            res++;
            right = x.second;
        }
    }
    return res;
}

    // [x] := 頂点 x にかかる遅延評価 (禁止する部分文字列の長さ)
    // 「*」の位置として許容できる箇所が狭まるので、min で集約する
    vector<int> lazy_op(T.tree_size(),s.size()+1);

    int q;
    cin >> q;
    for(int i = 0 ; i < q ; i++){
        string t;cin >> t;
        if(T.find(t) == false)continue;
        auto place = T.search(t);
        if(place.to == nullptr)lazy_op[place.from->id] = min(lazy_op[place.from->id], int(t.size()));
        else lazy_op[place.to->id] = min(lazy_op[place.to->id], int(t.size()));
    }

    // [i] := i 文字目以降で * が登場する必要がある箇所の制約を表す半開区間の右端
    vector<int> interval_right(s.size(),0);

    auto dfs = T.dfs();

            lazy_op[x->id] = min(lazy_op[x->id], lazy_op[x->parent->id]);
        }
        if(x->is_root())continue;
        if(x->isEndPoint()){
            interval_right[x->suffix_id] = x->suffix_id + lazy_op[x->id];

        intervals.push_back({i,interval_right[i]});
    }

    cout << RangeSchedulingProblem<int>(intervals) << endl;

/*

    - CharType 型の列 S に対して頂点 0 を根とする Suffix Tree を構築する。添字は基本的に 0-index。
        |- 列は string / vector の両方に対応。
            |- CharType は char からキャストされることができる必要があるものとする。
        |- Suffix Tree 上の各位置は S の連続部分列と 1 対 1 対応しており、実質的に連続部分列を重複なく列挙している。
        |- ノードの番号は生成順 (root の id が 0)。
        |- 空列も接尾辞として含み、辞書順で最小とする(suffix_id は |S| とする)
            |- 空列である接尾辞に対応するノードが root であるとする。
    |
    - Suffix Tree のデータのハンドルとして、Node と TreePlace クラスを内部に定義。詳細はクラスの中身を参照。
    |
    - Suffix Tree 構築
        |- push_back(s_) : S の末尾に列または文字 s_ をくっつけて Suffix Tree にも反映させる。
        |- determine() : 構築を終了する。
        |- build(s) := 初期状態から push_back(s) + determine()
    |
    - determine によって構築が終了することで、多くの機能が解放される。
        |- determine する前に使える機能もあるが少ない。
        |- determine する前に使える機能の説明の後には [(*)] を書いておく。
    |
    - 探索メソッド
        |- find(t) := t が S の連続部分列かどうかを判定。(*)
        |- count(t) := t が S の連続部分列として何回現れるか計算。
        |- search(t):= t を検索し、移動できた位置までのデータを返す。(*)
        |- search(l,r) := S の区間 [l,r) に対応する位置データを返す。ただし、refer は非明示的。
        |- decode_suffix(TreePlace x) := 位置 x に対応する列を連続部分列として含む接尾辞の開始 index のうちの 1 つを返す。(*)
            |- 返り値を l とすると、位置 x は S の [l,l + |x|) に対応する (|x| は位置 x に対応する列の長さ)
    |
    - 木構造メソッド
        |- Node クラスと TreePlace クラスは private なので、インターフェースを通して取得する。
            |- get_root() : 根ノードのポインタを返す。(*)
            |- from_root(bool refering = true) : 根に対応する位置を返す(位置データの refering を指定する)。(*)
            |- get_node(i) : 頂点番号が i のノードのポインタを返す。
        |- get_suffix_node(i) : S[i] から始まる接尾辞に対応するノードのポインタを返す。
        |- get_depth_as_tree(i) := root から頂点 i までの辺の本数を返す。
        |- edges() := Suffix Tree の辺集合を返す。(*)
        |- dfs(f) := root から dfs するときの到達順に頂点のポインタを並べて返す(f が true なら辞書順最小優先で dfs)。(*)
        |- LCA(位置のデータ 2 つ) := 位置(辺の途中含む)の LCA のデータを返す。
    |
    - その他機能
        |- クラス外で Suffix Tree 上の移動をする際は以下を使う
            |- findValidPath(p,c) := 位置から c の方向に移動できるかどうか。(*)
            |- walkdown(p,c) := 位置 p から c の方向に移動し、p を書き換える。(*)
        |- size() := S のサイズを返す。(*)
        |- [i] : S[i] を read-only でアクセス。(*)
        |- suffix_array() : Suffix Array を返す。
        |- get_Kth_subStr(K) : 列 S の 辞書順で K 番目の連続部分列。
    - 注意 
        |- release_resource を false にしておくと、リソースの解放がされない分高速になる。
        |- データアクセスは const Node のポインタを返すので、auto で受け取ると楽
*/
template<typename CharType, bool release_resource = true , typename = std::enable_if_t<std::is_convertible_v<char, CharType>>>
class SuffixTree{
    /*   
        Copyright © (c) okoteiyu 2024. 
        Released under the MIT license(https://opensource.org/licenses/mit-license.php) 
    */
    public:

    struct Node;
    struct TreePlace;
    using Edge = pair<CharType,Node*>; // {ラベルの先頭の文字, 終端のノード}
    using EdgeLabel = pair<int,int*>; // 半開区間による部分文字列の参照

    /* 頂点データ */
    struct Node{
        public:
        // 頂点番号
        const int id = -1;
        // [x] := 要素 x で続く辺の先のノード
        unordered_map<CharType , Node*> child;
        // 親ノード
        Node* parent = nullptr;
        // このノードまでのラベルからなる文字列の先頭を取り除いた文字列に対応する頂点へのリンク
        Node* SuffixLink = nullptr;
        // 自身に入る辺
        EdgeLabel label = make_pair(-1, nullptr);
        // このノードに対応する列の長さ (葉ノードについては不定なので、内部ノードにのみ定義)
        int path_label_length_sub = 0; 
        // 終端ノードなら何文字目から始まる接尾辞か(0-index)。そうでないなら -1。
        int suffix_id = -1; 
        // 自分以降の部分木が持つ終端ノードの個数
        int SubTreeEndPointCount = 0;      
        // このノードから移動して得られる文字列の種類数 (空文字列はカウントしない)
        long long SubTreeKindOfPrefix = 0; 
        // 葉頂点かどうか
        bool is_leaf = true;

        // ラベル、頂点番号、親頂点
        Node(EdgeLabel label_ , int id_, Node* parent_ , bool is_leaf_) 
            : id(id_), parent(parent_), label(label_), is_leaf(is_leaf_) {}

        Node()=default;
        Node(const Node&)=default;
        Node(Node&&)=default;
        Node& operator=(Node&&)=default;
        Node& operator=(const Node&)=default;

        // 辺に書かれた文字列の半開区間の左端 (0-index)
        const int cleft() const noexcept {
            return label.first;
        }
        // 辺に書かれた文字列の半開区間の右端 (0-index)
        const int cright() const noexcept {
            return (*label.second);
        }
        // 入る辺の長さ
        const int label_length() const noexcept {
            if(this->parent == nullptr)return 0;// root の場合は 0 とする
            return cright() - cleft();
        }
        // 根からノードまでの辺の文字数
        const int path_label_length() const noexcept {
            if(this->parent == nullptr)return 0;
            return this->parent->path_label_length_sub + label_length();
        }
        // このノードが終端ノードかどうか
        const bool isEndPoint() const noexcept {
            return (this->suffix_id >= 0);
        }
        // このノードが根かどうか
        const bool is_root() const noexcept {
            return bool(this->parent == nullptr);
        }
    };


    /*
        - (from, to, place) := 頂点 from から、to の方向に place 進んだ位置を指すデータ。
        - ちょうど 頂点 x の上である場合、(x, nullptr, 0) で統一する。
        - この位置に対応する列として refer を持つ。refering = false を指定すると refer を非明示的にして定数サイズオブジェクトとして扱える。
    */
    struct TreePlace{
        Node* from = nullptr;
        Node* to = nullptr;
        int place = 0;

        TreePlace()=default;
        // シャローコピー
        TreePlace(const TreePlace&)=default;
        TreePlace(TreePlace&&)=default;
        TreePlace& operator=(TreePlace&&)=default;
        TreePlace& operator=(const TreePlace&)=default;

        // 根から現在地までのパス上の文字数 (距離)
        int path_label_length() const noexcept {
            return this->from->path_label_length() + place;
        }
        // 根の位置かどうか
        bool is_root() const noexcept {
            return bool(this->path_label_length() == 0);
        }

        // この位置と対応する列を明示的に管理するかどうか
        bool refering = true;
        // この位置に対応する列 (pop_front するので deque で管理)
        deque<CharType> refer;
        
        // 現在地から、先頭要素を削除した位置に移動(明示的に対応する列 refer を管理する場合)。
        bool walkSuffixLink(){
            assert(refering);
            assert(refer.size() == this->path_label_length());
            if(this->is_root())return false; // これ以上 Suffix Link を辿れないなら false
            refer.pop_front();// 先頭の文字を削除する
            this->from = this->from->SuffixLink;// Suffix Link を辿って移動する
            // 辺上の位置を復元 (先頭を削除した refer の長さ -  直近の頂点への長さ)
            this->place = refer.size() - this->from->path_label_length();
            // 分岐点にいるなら終了して OK (place で判定する必要性に注意)
            if(this->place == 0){
                this->to = nullptr;
                return true;
            }

            // Suffix Link を辿ったあとの to を、from の深さと refer から復元
            this->to = this->from->child[refer[this->from->path_label_length()]];
            // place が from と to の間にくるように整形
            while(this->to != nullptr && this->to->label_length() <= this->place){
                this->place -= this->to->label_length(); 
                this->from = this->to;
                this->to = nullptr;
                if(this->place > 0)this->to = this->from->child[refer[this->from->path_label_length()]];
            }
            return true;
        }

        // refer が非明示な状況で Suffix Link を辿って移動する場合。
        // TreePlace は何らかの列の連続部分列なので、列 S_ と S_ のどこから始まる部分文字列かを表す整数を引数に取る。
        template<class ArrayType, typename = std::enable_if_t<
            is_same_v<typename decay<ArrayType>::type, string>
            || is_same_v<typename decay<ArrayType>::type, vector<CharType>>>>
        bool walkSuffixLink(const ArrayType& S_ , int start_index){
            refering = false;
            refer.clear();
            if(this->is_root())return false; 
            int refer_size = this->path_label_length()-1;// refer の先頭を削除したものの長さ
            start_index += 1;
            this->from = this->from->SuffixLink;
            this->place = refer_size - this->from->path_label_length();
            if(this->place == 0){
                this->to = nullptr;
                return true;
            }
            this->to = this->from->child[S_.at(start_index + this->from->path_label_length())];
            while(this->to != nullptr && this->to->label_length() <= this->place){
                this->place -= this->to->label_length(); 
                this->from = this->to;
                this->to = nullptr;
                if(this->place > 0)this->to = this->from->child[S_[start_index + this->from->path_label_length()]];
            }
            
            return true;
        }

        // 位置が同じ位置かどうか (refer を参照しないので O(1) 時間)
        friend bool operator==(const TreePlace& a,  const TreePlace& b){
            return (a.from == b.from && a.to == b.to && a.place == b.place);
        }
        
        // 出力
        friend ostream& operator<<(ostream& os , const TreePlace& p){
            os << "(" << p.from->id << ", ";
            if(p.to == nullptr)os << "?";
            else os << p.to;
            os << ", " << p.place << ")";
            return os;
        }
    };

    ** メンバ(これらは内部関数でのみ編集される) ============================================================
    ** ============================================================================================*/

    private:

    // エラー終了
    void build_error(){
        cerr << "Don't push_back to determined SuffixTree." << endl;
        exit(1);
    } 
    void calc_error(){
        cerr << "Cannot Calc from Not-Determined SuffixTree." << endl;
        exit(1);
    } 
    

    // 辺の順序(<)比較用のラムダ
    const function<bool(Edge,Edge)> compare_edge = [](Edge a ,Edge b){
        return a.first < b.first;
    };

    // 頂点番号割り振り用のカウンタ
    int m_vertex_id;
    // Suffix Tree 構築元の列
    vector<CharType> S;
    // S の終端を表すためのシンボル
    int m_EndS;
    // 根を表すデータ
    Node* root = nullptr;
    // 最後に追加された葉ノード
    Node* lastCreatedLeafNode = nullptr;
    // ノードを追加する際に、共有する部分の(端点の)位置データ
    TreePlace m_build_front;
    // [x] := 頂点番号 x のノードへのポインタ
    vector<Node*> m_Nodes;
    // [x] := x 文字目から始まる Suffix に対応するノードへのポインタ (0-index)

    // [x] := 根から頂点 x まで辺が何本か
    vector<int> m_tree_depth;
    // [i][u] := 頂点 u の 2^i 個上の頂点
    vector<vector<int> > m_parent; 
    /*== 重複して初期化を行わないためのフラグ ==*/
    // 構築が完全に終了したか。
    bool m_determined = false;
    // 基本的なデータを構築したか
    bool m_initialized_basic_data = false;
    // 部分木の集約データを構築したか
    bool m_initialized_tree_dp = false;
    // LCA のデータを構築したかどうか
    bool m_initialized_LCA = false;


    
    // メンバの初期化
    void initialize(){
        S.clear();
        m_Nodes.clear();
        m_SuffixNodes.clear();
        m_parent.clear();
        m_tree_depth.clear();

        m_determined = false;
        // 基本データを構築したか
        m_initialized_basic_data = false;
        // 部分木の集約データを構築したか
        m_initialized_tree_dp = false;
        // LCA データを構築したか
        m_initialized_LCA = false;
        // 構築用データ初期化
        m_vertex_id = 0;
        // 終端を表すシンボル
        m_EndS = 0;
        // 根頂点
        root = new Node({0,new int(0)}, m_vertex_id++, nullptr, false);
        // root からは root にリンクが出ている (共通化のためここだけ例外)
        root->SuffixLink = root; 
        // 最後に追加された葉ノード
        lastCreatedLeafNode = nullptr;
        // 位置データは refer を明示的に持つ
        m_build_front = from_root(true);
    }


    // 基本的なデータの構築(ノードへのアクセス用の配列など)
    void init_basic_data(){
        if(root == nullptr)return;
        if(m_initialized_basic_data)return;
        if(!m_determined)calc_error();
        
        m_initialized_basic_data = true;
        m_Nodes.resize(tree_size() , nullptr);
        m_tree_depth.resize(tree_size() , 0);
        m_SuffixNodes.resize(S.size()+1 , nullptr);// 空文字列 (|S| からの接尾辞) も含む
        vector<Node*> s = dfs();
        m_tree_depth[root->id] = 0;
        // DFS 順に実行
        for(Node* now : s){
            // id と Node* を対応付ける
            m_Nodes[now->id] = now;
            // x 文字目から始まる Suffix に対応するノードを管理
            if(now->isEndPoint())m_SuffixNodes[now->suffix_id] = now; 
            // 木としての深さを計算
            if(now->parent != nullptr)m_tree_depth[now->id] = m_tree_depth[now->parent->id] + 1;
        }
    }


    // 部分木の集約データの構築
    void init_tree_dp(){
        if(root == nullptr)return;
        if(m_initialized_tree_dp)return;
        if(!m_determined)calc_error();

        if(this->size() == 0){}
        m_initialized_tree_dp = true;
        vector<Node*> rdfs = dfs();
        reverse(rdfs.begin() , rdfs.end());
        for(Node* now : rdfs){   
            now->SubTreeKindOfPrefix = 0;
            now->SubTreeEndPointCount = 0;
        }
        for(Node* now : rdfs){
            if(now->isEndPoint())now->SubTreeEndPointCount+=1;
            if(now->parent == nullptr)continue;
            // 部分木のノードのうち、接尾辞と対応しているものの個数
            now->parent->SubTreeEndPointCount += now->SubTreeEndPointCount; 
            // 部分木のパスが表す文字列の種類数 + 辺の分
            now->parent->SubTreeKindOfPrefix += now->SubTreeKindOfPrefix + now->label_length();
        }
    }


    // LCA 用のデータの構築
    void init_LCA(){
        if(root == nullptr)return;
        if(m_initialized_LCA)return;
        if(!m_determined)calc_error();

        m_initialized_LCA = true;
        const int K = log2(tree_size())+4;
        m_parent.resize(K,vector<int>(tree_size(),-1));
        vector<Node*> s = dfs();        
        for(Node* now : s){
            if(now->parent == nullptr)continue;
            m_parent[0][now->id] = now->parent->id;
        }
        for(int x = 1 ; x < K ; x++){
            for(int u = 0 ; u < m_parent[x].size(); u++){
                if(m_parent[x-1][u] < 0)continue;
                m_parent[x][u] = m_parent[x-1][m_parent[x-1][u]];
            }
        }
    }


    // x をオンラインで S の末尾に連結する 
    void push_back_sub(CharType x){
        // 初回は必要データの構築を自動で行う
        if(root == nullptr || S.size() == 0)initialize();
        if(m_determined)build_error();
        
        S.push_back(x); 
        // 終端を表すシンボルの実体を更新
        m_EndS++;
        Node* lastUsedBranch = nullptr;
        do{
            // m_build_front → x に移動できるなら x の方向に移動して終了
            if(findValidPath(m_build_front,x)){
                if(lastUsedBranch!=nullptr)lastUsedBranch->SuffixLink = m_build_front.from; 
                walkdown(m_build_front,x);
                break;
            }
            // Branch を分岐点として S の接尾辞 {m_build_front + x} を Suffix Tree に挿入する
            Node* Branch = m_build_front.from;
            if(m_build_front.to != nullptr){// 辺の途中に分岐点を新たに作成する場合
                int middle = m_build_front.to->cleft() + m_build_front.place;
                Branch = new Node({m_build_front.to->cleft() , new int(middle) } ,m_vertex_id++ , m_build_front.from, false);
                // 辺の上に新たな Branch ができたことによる辺の張り替えなど
                m_build_front.from->child[ S[m_build_front.to->cleft()]] = Branch;
                Branch->parent = m_build_front.from;
                Branch->path_label_length_sub = m_build_front.path_label_length();
                m_build_front.to->label.first = middle;
                Branch->child[ S[m_build_front.to->cleft()] ] = m_build_front.to;
                m_build_front.to->parent = Branch;
            }
            // はみ出た部分 (1 文字分) を Leaf Node として追加する
            Node* NewLeaf = new Node( { m_EndS-1 , &m_EndS} ,m_vertex_id++ , Branch, true);
            NewLeaf->suffix_id = NewLeaf->cright() - m_build_front.path_label_length() - 1;
            Branch->child[x] = NewLeaf;

            if(lastUsedBranch!=nullptr)lastUsedBranch->SuffixLink = Branch;// 使った分岐点と leaf に Suffix Link を張る
            if(lastCreatedLeafNode != nullptr)lastCreatedLeafNode->SuffixLink = NewLeaf;
            
            lastUsedBranch = Branch;
            lastCreatedLeafNode = NewLeaf;
            // m_build_front を m_build_front + x に移動できるようになるまで、Suffix Link を辿る。
        }while(m_build_front.walkSuffixLink());
    }

    
    // 各接尾辞に対応するノードが Suffix Tree に存在するように変形する。
    void determine_sub(){
        if(m_determined)return;
        m_determined = true;
        // Suffix Link を辿りながら m_build_front の箇所にノードを生成する。
        do{
            Node* Branch = m_build_front.from;
            // 辺の途中に分岐点を新たに作成する場合
            if(m_build_front.to != nullptr){
                int middle = m_build_front.to->cleft() + m_build_front.place;
                Branch = new Node({m_build_front.to->cleft() , new int(middle) } ,m_vertex_id++ , m_build_front.from, false);
                m_build_front.from->child[ S[m_build_front.to->cleft()]] = Branch;
                Branch->parent = m_build_front.from;
                Branch->path_label_length_sub = m_build_front.path_label_length();
                m_build_front.to->label.first = middle;
                Branch->child[ S[m_build_front.to->cleft()] ] = m_build_front.to;
                m_build_front.to->parent = Branch;
            }
            // 根も空文字列に対応するものとして EndPoint になる。
            Branch->suffix_id = m_EndS - m_build_front.path_label_length();
            if(lastCreatedLeafNode!=nullptr)lastCreatedLeafNode->SuffixLink = Branch;// Suffix Link を張る
            lastCreatedLeafNode = Branch;
        }while(m_build_front.walkSuffixLink());
    }

        if(release_resource)clear();

        vector<Node*> pool = this->dfs(false);

    // 位置データ data から要素 x に向かって進むことができるかを判定
    bool findValidPath(const TreePlace& data , CharType x) const {
        // ノードの上にいる場合は、x 方向に辺があるかを確かめる
        if(data.to == nullptr )return bool(data.from->child[x] != nullptr);
        // 辺の上にいるときは、次の文字を確かめる
        return bool(S[data.to->cleft() + data.place] == x);
    }


    // 位置データ data から x の方向に 1 進み、data を更新する
    bool walkdown(TreePlace& data , CharType x) const {
        // 進めない場合
        if(findValidPath(data,x) == false)return false;
        // 分岐点にいる場合
        if(data.to == nullptr)data.to = data.from->child[x];
        data.place++;
        // 頂点に到達した場合、from が頂点を表す様にフォーマットする
        if(data.to != nullptr && data.place == data.to->label_length()){
            data.from = data.to;
            data.to = nullptr;
            data.place = 0;
        }
        // 位置データに対応する列が明示的なら、それを更新
        if(data.refering)data.refer.push_back(x);
        return true;
    }

    // 列 V_ を Suffix Tree 構成元の列の末尾に追加する
    template<class ArrayType, typename = std::enable_if_t<
        is_same_v<typename decay<ArrayType>::type, string>
        || is_same_v<typename decay<ArrayType>::type, vector<CharType>>>>
    void push_back(const ArrayType& V_){
        for(int i = 0 ; i < V_.size() ; i++)push_back_sub(CharType(V_[i]));
    }

    // 文字 c を Suffix Tree 構成元の列の末尾に追加する
    void push_back(CharType c){
        push_back_sub(c);
    }

    // 構築を終了する。
    void determine(){
        determine_sub();
    }

    // 列 V_ をもとに Suffix Tree を構築し、構築を終了する。
    template<class ArrayType, typename = std::enable_if_t<
        is_same_v<typename decay<ArrayType>::type, string>
        || is_same_v<typename decay<ArrayType>::type, vector<CharType>>>>
    void build(const ArrayType& S_){
        initialize();
        push_back(S_);
        determine();
    }

    // 列 X が、Suffix Tree の構成元の列 S の部分文字列として含まれるかどうか
    template<class ArrayType, typename = std::enable_if_t<
        is_same_v<typename decay<ArrayType>::type, string>
        || is_same_v<typename decay<ArrayType>::type, vector<CharType>>>>
    bool find(const ArrayType& X) const {
        TreePlace p_data = from_root(true);
        for(CharType x : X)if(!walkdown(p_data,x))return false;
        return true;
    }


    // 列 X が、Suffix Tree の構成元の列 S の部分文字列としていくつ含まれるか
    template<class ArrayType, typename = std::enable_if_t<
        is_same_v<typename decay<ArrayType>::type, string>
        || is_same_v<typename decay<ArrayType>::type, vector<CharType>>>>
    int count(const ArrayType& X){
        init_tree_dp();
        TreePlace p_data = from_root(true);
        for(CharType x : X)if(!walkdown(p_data,x))return 0;
        if(p_data.to == nullptr)return p_data.from->SubTreeEndPointCount;
        return p_data.to->SubTreeEndPointCount;
    }

    // Suffix Tree の根から列 X に沿って移動できるだけ移動し、最終的な位置データを返す
    template<class ArrayType, typename = std::enable_if_t<
        is_same_v<typename decay<ArrayType>::type, string>
        || is_same_v<typename decay<ArrayType>::type, vector<CharType>>>>
    TreePlace search(const ArrayType& X) const {
        TreePlace p_data = from_root(true);
        for(CharType x : X)if(!walkdown(p_data,x))break;
        return p_data;
    }


    /*
        Suffix Tree の元の文字列 S の [l,r) の部分文字列に対応する位置データを返す。
            |- 部分文字列に対応する定数サイズオブジェクトを O(logN) 時間で取得できておいしい。
            |- 返り値の位置データは、自身が表す文字列に関して非明示的 (refering が false)
            |- 長さ 0 の区間 [x,x) は空文字列に対応
    */
    TreePlace search(int l , int r ){
        // S の l 文字目から始まる接尾辞方向に (r-l) 文字進んだ位置が答え
        init_LCA();
        init_basic_data();
        assert(0 <= l && l <= r && r <= S.size());// 空文字列も OK
        TreePlace p_data = from_root(false);// (root,nullptr,0,{}) のデータ
        Node* border = m_SuffixNodes[l];
        
        // ギリギリ (r-l) 文字以上まで親方向に戻る (ダブリング)
        for(int k = int(m_parent.size())-1; k >= 0 ; k--){
            int y = m_parent[k][border->id];
            if(y < 0)continue;
            if(m_Nodes[y]->path_label_length() >= r-l)border = m_Nodes[y];
        }
        // さらに 1 つ戻ることで、S[l,r) に到達できる直近のノードになる
        if(border->path_label_length() > (r-l)){
            border = border->parent;
            p_data.to = border->child[S[l + border->path_label_length()]];
            p_data.place = (r-l) - border->path_label_length();
        }
        p_data.from = border;
        return p_data;
    }


    // 頂点 x の位置に対応する文字列を含む接尾辞の開始位置を返す (0-index)
    // 複数存在する場合はいずれか 1 つを返す
    int decode_suffix(const Node* x) const {
        assert(x != nullptr);
        if(x == root)return m_EndS;
        return x->cleft() - x->parent->path_label_length();
    }   

    // 位置 p_data に対応する文字列を含む接尾辞の開始位置を返す (0-index)
    // 複数存在する場合はいずれか 1 つを返す
    int decode_suffix(const TreePlace& p_data) const {
        if(p_data.to!=nullptr)return decode_suffix(p_data.to);
        return decode_suffix(p_data.from);
    }


    // 頂点番号で普通にLCA
    int LCA(int u , int v){
        init_LCA();
        init_basic_data();
        assert(0 <= u && u < tree_size());
        assert(0 <= v && v < tree_size());
        if(m_tree_depth[u] > m_tree_depth[v])swap(u,v);
        for(int k = int(m_parent.size())-1 ; k >= 0 ; k--){
            if(m_parent[k][v] < 0)continue;
            if(m_tree_depth[m_parent[k][v]] < m_tree_depth[u])continue;
            v = m_parent[k][v];
        }
        if(u == v)return u;
        for(int k = int(m_parent.size())-1 ; k >= 0 ; k--){
            if(m_parent[k][u] == m_parent[k][v])continue;
            u = m_parent[k][u];
            v = m_parent[k][v];
        }
        return m_parent[0][u];
    }


    // ポインタで示された頂点 u と v の LCA。返り値もポインタ。
    Node* LCA(const Node* u, const Node* v){
        assert(u != nullptr);
        assert(v != nullptr);
        // 同じ木の頂点かを確認
        assert(get_node(u->id) == u);
        assert(get_node(v->id) == v);
        return m_Nodes[LCA(u->id , v->id)];
    }

    // 位置データ sd と、頂点ポインタ v の LCA。
    // LCA が 辺の途中になる可能性もあるので TreePlace で返す。
    TreePlace LCA(const TreePlace& sd, const Node* v){
        // 位置 sd が根から v までのパス上にある場合
        if(sd.to!=nullptr && LCA(sd.to , v) == sd.to)return sd;    
        TreePlace res = from_root();
        res.from = LCA(sd.from , v);
        return res;
    }

    // 位置データ sd と、頂点ポインタ v の LCA。
    // LCA が 辺の途中になる可能性もあるので TreePlace で返す。
    TreePlace LCA(const Node* v , const TreePlace& sd){return LCA(sd,v);}

    // 位置データ sd と、位置データ sp の LCA。
    // LCA が 辺の途中になる可能性もあるので TreePlace で返す。
    TreePlace LCA(TreePlace sd ,TreePlace sp){
        TreePlace res = from_root();
        if(sd.to != nullptr && sp.to != nullptr){
            if(sd.to == sp.to && sd.from == sp.from){// 同じ辺の上の位置の場合
                if(sd.path_label_length() < sp.path_label_length())return sd;// 浅い方が答え
                return sp;
            }
            // 同じ辺にいない場合、深い方は from まで下げて良い
            if(sd.path_label_length() > sp.path_label_length()){
                // sp が深い方で標準化
                TreePlace temp = sd;
                sd = sp;
                sp = temp;
            }
            return LCA(sd,sp.from);
        }else if(sd.to == nullptr && sp.to != nullptr)return LCA(sp , sd.from);
        else if(sp.to == nullptr && sd.to != nullptr)return LCA(sd , sp.from);
        res.from = LCA(sd.from , sp.from);
        return res;
        
    }

    // root を表す位置データを返す。
    // refering が false なら対応する列が非明示的な位置データを作って返す。
    TreePlace from_root(bool refering = true) const {
        TreePlace res;
        res.from = root;
        res.to = nullptr;
        res.place = 0;
        res.refering = refering;
        res.refer = {};
        return res;
    }

        assert(0 <= v && v < tree_size());
        return m_Nodes[v];
    }

    // i 番目の suffix の終端に対応するノードを返す(0-index)
    const Node* get_suffix_node(int i){
        init_basic_data();
        assert(0 <= i && i < m_SuffixNodes.size());
        return m_SuffixNodes[i];
    }

    // 根ノードを返す
    const Node* get_root() const {
        return root;
    }

    // 頂点 v の深さ(根から辺を何本経由するか)
    int get_depth_as_tree(int v){
        init_basic_data();
        assert(0 <= v && v < tree_size());
        return m_tree_depth[v];
    }

    // 構築元の列 S の S[i] を返す (0-index で read-only)

        assert(0 <= i && i < S.size());
        return S[i];
    }

    // 構築元の列 S の S.size()
    const int size() const {
        int res = S.size();
        return res;
    }

    // 構築元の列 S の連続部分列の種類数。空文字列も含む。
    long long KindOfSubSequence(){
        init_tree_dp();
        // 空文字列は DP でカウントされないのでここで加算
        return root->SubTreeKindOfPrefix + 1;
    }

    // Suffix Tree の頂点数
    const int tree_size() const {
        return m_vertex_id;
    }

    // 木の辺を頂点番号のペアとして、辺集合を vector で返す
    vector<pair<int,int>> edges() const {
        vector<Node*> dfs_ = dfs(false);
        vector<pair<int,int>> res;
        for(Node* x :dfs_){
            if(x->parent==nullptr)continue;
            res.push_back({x->parent->id , x->id});
        }
        return res;
    }

    // dfs 順の Node* を返す (辞書順に dfs するかを引数で指定)
    vector<Node*> dfs(bool lexicographical = false) const {
        vector<Node*> res;
        if(root == nullptr)return res;
        stack<Node*> s;
        s.push(root);
        while(!s.empty()){
            Node* now = s.top();s.pop();
            res.push_back(now);
            vector<Edge> nxt;
            for(Edge nx : now->child)if(nx.second != nullptr)nxt.push_back(nx);
            // 辺の最初の文字でソート
            if(lexicographical)sort(nxt.rbegin() , nxt.rend() , compare_edge);
            for(Edge nx : nxt)s.push(nx.second);
        }
        return res;
    }


    /* ==============================================================================================
    ** 内蔵している典型問題たち =========================================================================
    ** ============================================================================================*/
    
    /*
        このクラスが保持する列の Suffix Array を返す (0-index)
            |- 空列は辞書順最小の接尾辞とし、その suffix_id を |S| とする
    */
    vector<int> suffix_array(){
        if(!m_determined)calc_error();
        vector<Node*> s = dfs(true);
        vector<int> res;
        for(Node* now : s){
            if(!now->isEndPoint())continue;
            res.push_back(now->suffix_id);
        }
        return res;
    }
    

    /*
        列 S の 辞書順で K 番目の連続部分列
            |- 返り値は vector<CharType>
            |- K = 0 なら空の列を返す
    */
    vector<CharType> get_Kth_subSeq(long long K){
        init_tree_dp();
        assert(root != nullptr);
        assert(0 <= K && K < KindOfSubSequence());
        
        Node* now = root;
        long long acum = 0;
        vector<CharType> res(0);
        if(K == 0)return res;

        while(K > acum){
            vector<Edge>cand;
            for(Edge pr : now->child)if(pr.second!=nullptr)cand.push_back(pr);
            // 辺の最初の文字の昇順でソート
            sort(cand.begin(),cand.end(),compare_edge);

            for(Edge pr : cand){
                Node* nx = pr.second;
                int nx_edge_num = nx->label_length();// 次の頂点への辺上のprefixの種類数
                // 辺の先に答えがある場合
                if(nx->SubTreeKindOfPrefix + nx_edge_num + acum >= K){
                    if(acum + nx_edge_num >= K){// 答えとなる文字列が、次の頂点への辺の上にあるとき
                        for(int i = nx->cleft() ; i < nx->cleft() + K - acum ; i++ )res.push_back(S[i]);
                        return res;
                    }else{
                        for(int i = nx->cleft() ; i < nx->cright() ; i++ )res.push_back(S[i]);
                        now = nx;// 辺の先に移動 
                        acum += nx_edge_num;
                        break;// さっきいたノードからは抜ける
                    }
                }else{
                    acum += nx->SubTreeKindOfPrefix + nx_edge_num;// この辺の先に答えはないので、次の辺を見る
                } 
            }
        }
        assert(0);
        return res;
    }
