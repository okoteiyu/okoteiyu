有理数型

有理数クラス

C++ で分数を数値計算するとどうしても誤差がでます。また \(\infty , \varepsilon\) などの特殊な値を表現することができません。そこで、有理数クラスを自作しました。

概要

整数型 (__int128_t などのオーバーフローしづらいもの) の \(A,B\) に対して有理数 \(\frac{A}{B}\) を表現します (普通にオーバーフローしやすいから注意)。

  • 値を \(\mathrm{T}\) 型で取り出したい場合は val() を呼び出すか、\(\mathrm{T}\) 型に明示的にキャストする (\(\mathrm{T}\) が整数ならあまりは切り捨て)。
  • \(A,B\) を取り出したいときは raw_pair() でアクセス。

Fraction がとる値

冒頭で説明した通り、Fraction は整数 \(A,B\) で \(\frac{A}{B}\) の形で定義されます。Fraction は通常の数値に加え、無限大/微少量 などの値をとりうる特殊な型です。特殊な値 (インスタンス) は以下の通りです。

  • \(\mathrm{amb} := \frac{0}{0}\) (ゼロ徐算や無限大同士の減算など)
  • \(\mathrm{inf} := \infty\) (無限大)
  • \(\varepsilon := \frac{1}{\infty}\) (微少量)
\(\mathrm{amb},\mathrm{inf}\) は特殊インスタンスとして明示的に宣言/定義しますが、\(\varepsilon\) は \(\mathrm{inf}\) から導出されるので明示的に宣言/定義しません。

表現と計算

実数有理数の計算であれば、ただやるだけです。Fraction クラスのメインは、先に定義した \(\mathrm{amb},\mathrm{inf},\varepsilon\) に対して計算規則をうまく決めて、矛盾がないような 有理数+特殊インスタンス の計算を実現することです。

Fraction クラスによる有理数 + \(\alpha\) の表現は、メンバ \(A,B\) および補助的なメンバ \(\mathrm{eps\_sign}\) によって以下で定められます。

  1. \((A,B) = (0,0)\) の場合
    • \(\frac{A}{B} = \) 不定値 (\(\mathrm{amb}\)) として定義する。
  2. \((A,B) = (0,1)\) の場合、\(\mathrm{eps\_sign}\) というメンバの値で場合分けする。
    • \(\mathrm{eps\_sign} = 0\) なら \(\frac{A}{B} = 0\)
    • \(\mathrm{eps\_sign} = 1\) なら \(\frac{A}{B} = \varepsilon\)
    • \(\mathrm{eps\_sign} = -1\) なら \(\frac{A}{B} = -\varepsilon\)
  3. \((A,B) = (\pm 1 ,0)\) の場合、\(A\) の符号で場合分けする。
    • \(A = 1\) なら \(\frac{A}{B} = \infty\)
    • \(A = -1\) なら \(\frac{A}{B} = -\infty\)
    \(B=0\) かつ \(|A|\) が \(0\) 以外なら、符号だけ保存して \(|A| = 1\) の形にフォーマットする。
  4. 上記の \(1,2,3\) のいずれでもない場合、\(\frac{A}{B}\) は実数有理数を表す。ただし符号は \(A\) につける方針で統一する。

計算規則

Fraction オブジェクト \(x,y\) に対して、数学的に矛盾がないように四則演算を定義します。結果が \(\mathrm{amb}\) になる場合の計算さえ分かれば、あとは数学の規則に則って計算するだけです。

  1. \(\mathrm{amb}\) に対してどの演算を行っても結果は \(\mathrm{amb}\)
  2. \(x + y\) の計算
    • \(\infty\) と \(-\infty\) が混在するなら \(x + y = \mathrm{amb}\)
    • \(\varepsilon\) と \(-\varepsilon\) が混在するなら \(x + y = \mathrm{amb}\)
  3. \(x \times y\) の計算
    • 片方が \(\pm \varepsilon\) で他方が \(\pm \infty\) なら \(x \times y = \mathrm{amb}\)
  4. \(x-y\) の計算は \(x+(-y)\) の計算と等しいです。\(-y\) は \(y\times (-1)\) です。
  5. \(\frac{x}{y}\) は \(x \times y^{-1}\) なので \(y\) の逆数が分かればよく、\(y\) の逆数は以下の計算規則で計算可能です。
    • \(y = 0\) または \(y = \mathrm{amb}\) なら \( y^{-1} =\mathrm{amb}\)
    • \(y = \pm\varepsilon \) なら符号に合わせた \(y^{-1} =\infty\)
    • \(y = \pm\infty \) なら符号に合わせた \(y^{-1} =\varepsilon\)
    • 上記のいずれでもないなら \(y\) のメンバ \(A,B\) をスワップし、符号を \(A\) につけたものが \(y^{-1}\)。
ただし、\(\varepsilon - \varepsilon + \infty\) は明らかに \(\infty\) ですが、この計算規則ではこれを計算できません。この点には注意が必要です。

設計

キャストについて

  • 自作クラスの機能を優先させたいので、C++ 標準型 \(\rightarrow\) Fraction のキャストは暗黙的。
  • 当てはまる演算子オーバーロードなどが複数存在する状況を避けるため、Fraction \(\rightarrow\) C++ 標準型 のキャストは明示的。

メンバの持ち方について

以下の制約を満たすように \(A,B\) を整形する format 関数を用意し、コンストラクタなどでの初期化時に呼び出す。演算などを行うたびに Fraction オブジェクトを生成し、コンストラクタを通すことで常に format されるようにする。

  • \(\frac{A}{B}\) が有理数の場合、\(\frac{A}{B}\) は常に既約であるように設計 (gcd で割る)
  • 符号は \(A\) につける。分母 \(B\) は常に非負整数。

比較演算

三方演算子 std::partial_ordering <=>(T a , T b) を使います (C++ 20 からの機能)。

  • <=> の実装が返す以下の結果から、< , <= , > , >= 演算の結果を作るすごいやつ。
    • std::partial_ordering::equivalent : a と b が等しい
    • std::partial_ordering::less : a が b 未満
    • std::partial_ordering::greater : a が b より大きい
    • std::partial_ordering::unordered : a と b の順序が未定義
  • 例えば、0 <=> ∞ が less を返すように実装する。すると、0 < ∞ の結果が true になる。
  • unordered について
    • a,b の順序比較ができない時、unordered を返し、全ての順序演算が false を返す(無限大同士の比較など)。
      • unordered な順序があるバブルソートは永遠に終わらない
      • unordered な順序がある列をソートすると、unordered な順序以外の要素も正しくソートされない
    • できるだけ unordered な順序がある要素を比較しない

その他

  • 各演算の両辺は必ず整数 or Fraction
  • 演算の規則は一般の数学の規則に準拠
  • コンストラクタで明示的に分子が非ゼロ、分母がゼロを与えた場合、それは無限大/無限小 になる。

class Fraction{
    /*   
        Copyright ©️ (c) okoteiyu 2024.

    protected:
    /*
        内部用のメンバ。!!!! 依存関係が内部用メンバ関数のみで完結 !!!!
    */
    using Integer = __int128_t;// 内部でもつ整数
    Integer A,B;
    // (ε なら 1 , -ε なら -1, 0 なら 0)
    Integer m_eps_sign = 0;// デフォルト (A == 0 の時のみアクセス可能)
    Integer gcd(Integer x, Integer y) { return (y!=0) ? gcd(y, x % y) : x; }

    // A ,B の形を制約通りに整える
    void format(){
        if(A == 0 && B == 0){}// 0/0 は未定義
        else if(A == 0)B = 1; // A が 0 の時、B は 1 (微少量かどうかは m_eps_sign を見る)
        else if(B == 0){// ∞ , -∞ の時、A は 1 or -1 とする
            Integer sign_ = 1;
            if(this->A < 0)sign_ = -1;
            this-> A = sign_;
        }else{ // 有理数として評価できる場合
            // A/B は規約分数。B は常に正整数 (非零) とする
            Integer g = gcd(A,B);
            A/=g;B/=g;
            if(B < 0){A*=-1;B*=-1;} 
        }
    }

    public:
    // 特殊インスタンス
    inline static const Fraction inf(){return Fraction(1,0);}// 無限大
    inline static const Fraction ninf(){return Fraction(-1,0);}// 無現小
    inline static const Fraction amb(){return Fraction(0,0);}// 0/0
    inline static const Fraction zero(){return Fraction(0,1);}// 0
    /*
        コンストラクタやコピー。
        依存先が内部メンバのみの最も基本的な関数
    */
    Fraction(){}
    // A,B に入る値は 64 bit 整数じゃないと演算内でオーバーフローする
    // Integer to Fraction ()
    Fraction(Integer a):A(a),B(1),m_eps_sign(0){this->format();}// 通常の整数型からのキャスト用
    // Fraction
    Fraction(Integer a , Integer b):A(a),B(b),m_eps_sign(0){this->format();}
    Fraction(const Fraction& x):A(x.A),B(x.B),m_eps_sign(x.m_eps_sign){this->format();}
    Fraction& operator =(const Fraction& x){
        this->m_eps_sign = x.m_eps_sign;
        this->A = x.A;
        this->B = x.B;
        this->format();
        return (*this);
    } 

    /*
        補助機能。外部でも使う。
    */

    // Fraction 型の絶対値
    Fraction abs() const& {return (*this)*this->sign();}

    // 0/0 かどうか
    bool is_ambiguous() const& {return bool(this->A == 0 && this->B == 0);}
    // ∞ または -∞ かどうか
    bool is_infinite() const& {return bool(this->B==0);}
    // 符号 (負なら -1 , 正なら 1 ,0 なら 0)
    // is_ambiguous の場合は エラー (注意)
    Integer sign() const& {
        assert(!this->is_ambiguous());
        if(this->A == 0)return this->m_eps_sign;;// 0 or ±ε の場合
        if(this->A < 0)return -1;
        return 1;
    }

    /*
        アクセス/キャスト
    */
    
    // 実数型にして返す関数 (整数だと切り捨てが発生してしまうのでだめ)
    template<typename real>constexpr real to_real() const& {return real(A)/real(B);}
    // {A,B} のペアを返す
    pair<Integer,Integer> raw_pair() const& {return make_pair(A,B);}
    //
    // 実姿へのキャスト (明示的)。別のクラスに入れる場合は、そのクラスのキャスト処理内で明示的にする。詳細は okoteiyu ライブラリ設計規則
    // (整数だと切り捨てが発生してしまうのでだめ)
    // 
    template<typename real>explicit operator real() const& {return real(A)/real(B);}
    //
    // 逆数を返す
    Fraction inv() const& {
        if(this->is_ambiguous())return amb();// 曖昧なやつはそのまま
        if(this->A == 0){
            if(this->sign() == 0)return amb();// 0 で割るのは曖昧
            return Fraction(this->sign(),0);// 微少量は、同じ符号の無限大に
        }
        if(this->B == 0){// 無限大は同じ符号の微少量に
            Fraction res(0,1);
            res.m_eps_sign = this->sign();
            return res;
        }
        return Fraction(this->B,this->A);// 逆数
    }
    //
    // 代入演算たち
    //      |- primary な演算は + と * で、これらは他の Fraction 型演算に依存しない。
    //
    
    // たし算。他の演算に依存してはいけない
    Fraction& operator +=(Fraction x){
        if(this->is_ambiguous() || x.is_ambiguous()) [[unlikely]] (*this) = amb();// 0/0 は変わらない
        else if(this->is_infinite() && x.is_infinite()) [[unlikely]] {// 両方とも無限大/小 の場合 
            if(this->sign() != x.sign())(*this) = amb();// ∞ - ∞ は曖昧
            else if(this->sign() == -1)(*this) = ninf();// 符号がどちらもマイナス
            else (*this) = inf(); // 符号がどちらもプラス
        }else if(this->is_infinite()) [[unlikely]] {
            // 代入される側のみ無限大の場合、何も代入しなくて OK
        }else if(x.is_infinite()) [[unlikely]] (*this) = x;// 代入する側が無限大ならそのまま代入
        else if(this->A == 0 && x.A == 0) [[unlikely]] {// 両方とも 0 or ±ε の場合 
            if(this->sign() == x.sign()){}// 同じ符号 (または両方 0) ならそのまま
            else if(this->sign() == 0)(*this) = x;// 片方 0 なら、0 じゃない方に合わせる
            else if(x.sign() == 0){}// 片方 0 なら、0 じゃない方に合わせる
            else (*this) = amb();// 反対の符号なら曖昧になる
        }else [[likely]]{
            // (0 or 微少量は 0/1 の形なので、片方が 0 でも 微少量でもない時はそのまま計算可能)
            (*this) = Fraction(this->A*x.B + this->B*x.A , this->B*x.B);
        }
        return (*this);
    }
    // かけ算。他の演算に依存してはいけない
    Fraction& operator *=(Fraction x){
        if(this->is_ambiguous() || x.is_ambiguous()) [[unlikely]] (*this) = amb();
        else if(this->is_infinite() || x.is_infinite()) [[unlikely]] {// どちらかが無限の場合
            if(this->A != 0 && x.A != 0){// どちらも 0 でも 微少量でもない場合
                if(this->sign() == x.sign())(*this) = inf();// 符号に依存する
                else (*this) = ninf();
            }else{// どちらかが 0 or 微少量の場合
                if(this->sign() == 0 || x.sign() == 0)(*this) = 0;// 0 にかけたらなんでも 0 になる
                else (*this) = amb(); // ±ε × 無限 は is_ambiguous
            }
        }// 以降,両方とも無限ではない
        else if(this->A == 0 || x.A == 0) [[unlikely]] {// 少なくとも一方が 0 or ±ε の時
            if(this->sign() == 0 || x.sign() == 0)(*this) = 0;// 0 にかけたらなんでも 0 になる
            else {
                Fraction new_eps = 0;// 微少量になる場合。片方は ε でも 0 でもないかもしれないので sign を参照
                new_eps.m_eps_sign = this->sign()*x.sign();
                (*this) = new_eps;
            }
        }else [[likely]] {(*this) = Fraction( this->A*x.A, this->B*x.B );}// 実数
        return (*this);
    }
    // - の値 (* 演算に投げる。)
    Fraction operator -(){Fraction res(*this);return res*Fraction(-1);}
    // 減算 ( + に投げる)
    Fraction& operator -=(Fraction x){return ((*this) += (-x));}
    // 割り算は inv を使う
    Fraction& operator /=(Fraction x){return ((*this) *= x.inv());}
    // 徐算 (有理数で割っても OK な特殊なやつ)
    // 整数 m で (this) = m*x + r と表せる r を返す (返す r の規則は int の % と同じ挙動)
    Fraction& operator %=(Fraction x){
        // 収束した値同士でないといけない
        assert(!this->is_ambiguous() && !this->is_infinite() && !x.is_ambiguous() && !x.is_infinite());
        assert(x.A!= 0);// 0 徐算はだめで、ε も不定なのでだめ
        Fraction tmp = (*this);
        tmp/=x;
        Integer m = tmp.raw_pair().first/tmp.raw_pair().second;// m は整数に丸める
        (*this) -= m*x;
        return (*this);
    }

    // 二項演算。整数 → Fraction のキャストは暗黙なので、これらの演算は、片方が整数でも OK。
    friend Fraction operator +(Fraction a , Fraction b){return a+=b;}
    friend Fraction operator -(Fraction a , Fraction b){return a-=b;}
    friend Fraction operator *(Fraction a , Fraction b){return a*=b;}
    friend Fraction operator /(Fraction a , Fraction b){return a/=b;}
    // 徐算 (有理数で割っても OK な特殊なやつ) (収束している値同士である必要がある)
    friend Fraction operator %(Fraction a , Fraction b){return a%=b;}
    // 
    // 比較演算たち 
    /*  
        - <=> が返す less / greater / equivalent の結果から、< , <= , > , >= 演算の結果を作るすごいやつ
        - 順序が定義できるものにしか使えない。unordered を返すケースではソートなどがうまくいかないので注意。
    */
    friend partial_ordering operator <=>(const Fraction& x , const Fraction& y){
        // どちらかが 0/0 なら比較は不可能。プログラム内部での値も異なるので、unordered 
        if(x.is_ambiguous() || y.is_ambiguous()) [[unlikely]] return partial_ordering::unordered;
        // 符号が異なる場合、符号で判定して OK . (符号は -1,0,1 のいずれか)
        else if(x.sign() != y.sign()) [[unlikely]] return (x.sign() <=> y.sign());
        // ここからは、符号は同じとして OK 
        else if(x.A == 0 || y.A == 0) [[unlikely]] {// いずれかが 0 または 微少量の場合
            if(x.sign() == 0 && y.sign() == 0)return partial_ordering::equivalent;// 0 同士
            // 両方とも微少量の場合 (本来比較は無理だが、プログラム内部では同じなので、equivalent として OK)
            if(x.A == 0 && y.A == 0)return partial_ordering::equivalent; 
            else if(x.A == 0){// x のみ微少量なら、y の符号で決める
                if(y.sign() == -1)return partial_ordering::greater;
                return partial_ordering::less;
            }else{// y のみ微少量の場合、x の符号で決める
                if(x.sign() == -1)return partial_ordering::less;
                return partial_ordering::greater;
            }
        }
        // 以降、どちらも 0 でも微少量でもないとして OK ( & 符号も同じ )
        // 符号が同じ無限大の比較は不可能。しかし、プログラム内部では同じなので equivalent として OK
        else if(x.is_infinite() && y.is_infinite()) [[unlikely]] return partial_ordering::equivalent;
        else if(x.is_infinite()) [[unlikely]] {// x のみ無限大 or 無限小
            if(x.sign() == -1)return partial_ordering::less;
            return partial_ordering::greater;
        }else if(y.is_infinite()) [[unlikely]] {// y のみ無限大 or 無限小
            if(y.sign() == -1)return partial_ordering::greater;
            return partial_ordering::less;
        }
        else [[likely]] return ((x.A)*(y.B) <=> (y.A)*(x.B));
    }

    /*0 周りの挙動が特殊なので注意が必要*/
    friend bool operator ==(const Fraction& x,const Fraction& y){
        // 収束していない値同士を比較できない。この場合、false を返すものとする。
        if(x.is_ambiguous() || x.is_infinite() || y.is_ambiguous() || y.is_infinite())return false;
        if(x.A == 0 && y.A == 0){// (0 or 微少量) 同士を比較する
            if(x.sign() == 0 && y.sign() == 0)return true;// 0 同士のみ true
            return false;// 符号が同じ ε 同士は曖昧な関係 & ε != 0 & ε != -ε
        }
        // 0 も 微少量も 0/1 の形なので、片方が有理数なら単純比較して OK
        return bool(x.A == y.A && x.B == y.B);
    }
    friend bool operator !=(const Fraction& x,const Fraction& y){return !(x==y);}

        os.precision(40);
        if(x.is_ambiguous())os << "?";
        else if(x.A == 0){
            if(x.sign()<0)os << "-ε";
            else if(x.sign()>0)os << "ε";
            else os << 0;
        }else if(x.is_infinite()){
            if(x.sign() < 0)os << "-∞";
            else os << "∞";
        }else os << double(x.A) << "/" << (long long)(x.B);
        return os;
    }

        is >> tmp;

        return is;
    }
