ライブラリに載せる演算

特定のデータ構造やアルゴリズムのライブラリを作るとき、そのライブラリが行う処理を抽象化して書く場合を想定します。

例えばセグメント木は区間和、区間 min など様々なクエリを処理することができますが、クエリの中身を抽象化しておけば 1 つのセグ木ライブラリで全てのクエリに対応することができます。


二項演算について



演算とは

二項演算を \((S , \cdot )\) のように表記します。「\(\cdot\)」 は演算子を意味し、以下を満たします。
  • \(a,b \in S\) に対して、\(a \cdot b \in S\)
    • 例えば、整数の足し算は \((\mathbb{Z} , +)\) と表記します (\(\mathbb{Z}\) は整数の集合)。

    ライブラリに載せることができる演算はライブラリごとに制限があります。まずはよく使う以下の 2 種類を説明します。

    群

    以下の要件を満たす演算 \((S , \cdot )\) を群といいます。
    • \(\forall a,b,c \in S\) に対して \((a\cdot b)\cdot c = a\cdot (b\cdot c)\) (結合法則)
    • 以下を満たす \(e \in S\) が存在する (単位元)
      • \(\forall x \in S\) に対して \((x\cdot e) = (e\cdot x) = x\)
    • \(\forall x \in S\) に対して以下を満たす元 \(r\) が存在する (\(x\) の逆元)
      • \((r \cdot x) = (x \cdot r) = e\) (\(e\) は単位元)
    例えば、整数の足し算 \((\mathbb{Z} , +)\) は群です。
    • \((a + b) + c = a + (b + c) \dots 結合法則 \)
    • \(\forall x \in S : x + 0 = 0 + x = x \dots (0 は単位元) \)
    • \(\forall x \in S : x + (-x) = (-x) + x = 0 \dots (x の逆元) \)

    ただし群は非可換かもしれないことに注意してください。例えば正則行列の積 \((M,\cdot)\) は非可換な群です。\( A,B \in M\) に関して \(A\cdot B\) は必ずしも \(B\cdot A\) と一致しません。

    可換であることが保証される群を可換群またはアーベル群と呼びます。



    半群(モノイド)

    群の要件から逆元を無くした演算を半群またはモノイドと呼びます。
    • \(\forall a,b,c \in S\) に対して \((a\cdot b)\cdot c = a\cdot (b\cdot c)\) (結合法則)
    • 以下を満たす \(e \in S\) が存在する (単位元)
      • \(\forall x \in S\) に対して \((x\cdot e) = (e\cdot x) = x\)
    例えば整数の掛け算 \((\mathbb{Z} , \times )\) はモノイドですが群ではありません。
    • \((a \times b) \times c = a \times (b \times c) \dots 結合法則 \)
    • \(\forall x \in S : x \times 1 = 1 \times x = x \dots (1 は単位元) \)
    • \(0\) の逆元を定義できない
    こちらも非可換かもしれないことに注意してください。

    以下の主要なライブラリに載る演算もこれらでカテゴライズできます。ライブラリに載せる演算を設計する際は、これらの要件を満たすかをよくチェックする必要があります。

    (両者の {可換/非可換} が一致する場合) モノイドと群ではモノイドの方が要件がゆるく、適用できる範囲が広いため、同じアルゴリズムのライブラリでも群よりモノイドが乗る方が優秀です。


    セグメント木

    分割統治区間の必要な部分だけ演算するので逆演算の必要がなく、列なので要素の順番が規定されているため可換である必要もありません。よってセグメント木にはモノイドが載ります。


    Splay Tree

    セグ木と同様、その都度必要なデータを演算するのでモノイドが載ります。


    全方位木 DP

    部分木のマージの順番が定義されているならモノイドが載ります。普通は可換モノイドが載ります。

    要求が可換群で良いならなら実装を楽することができます。


    UnionFind + 連結成分の要素の演算の積

    それぞれの頂点に価値 \(v_i\) が割り当てられる状況で Union Find Tree を構築するような場合です。

    \(v\) の更新がなければ可換モノイドが載ります。

    \(v\) の更新があるなら逆演算(逆元)が必要なので可換群(アーベル群)が載ります。




    遅延評価について

    載せたい遅延評価を抽象化することもできます。

    まずは遅延評価が載る条件をまとめます。

    • 1 つの値に集約できる (演算である)
    • 結合法則がある
    • 単位元がある
    つまり遅延評価もモノイドであることが条件となります。
