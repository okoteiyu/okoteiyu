フローと最適化

フローと最適化はとても深い関係にあります。
以下のスライドは、フローで解ける基礎的な最適化の問題の一例です。
























上のスライドで少し登場した最大流が今後のキモとなる概念です。スライドでは特に深い説明をしませんでしたが、 改めて最大流ないしネットワークフローが何者なのかを説明します。

辺重み付き有向 グラフ \(G = (V,E)\) および \(s,t \in V\) があります。以降、辺重みのことを辺の容量と呼びます。
「\(G\) における \(s \rightarrow t\) の流量 \(x\) のフロー」を以下で定義します。

  • 容量が \(x\) 以上の辺のみからなる \(s \rightarrow t\) パス
また、「\(s \rightarrow t\) にフローを \(x\) 流す」という操作を以下で定義します。
  • 容量が \(x\) 以上の辺のみからなる \(s \rightarrow t\) パスを \(1\) つ見つけて、パス上の全ての辺の容量を \(x\) 減らす。

最大流とは、「\(s \rightarrow t\) にフローを一度に流せるだけ流す」操作を限界まで行った結果流れた流量の合計です。

最大流の計算はスライドで軽く紹介した通り、フローが流れた辺と逆向きに 流れた分の容量をもつ逆辺を追加しながら、\(s \rightarrow t\) にフローを流せなくなるまで流し続けます。 ここで、流すフローの流量はどのように決めるのでしょうか。

以下のようなグラフでのネットワークフローを考えてみましょう。


流すフローの流量は、ある \(s \rightarrow t\) パスの容量のボトルネックとすれば良いです。例えば、以下のパスにフローを流す場合の流量を考えます。


このとき、流すフローの流量を容量のボトルネックである \(3\) とすれば、選んだパスに対して過不足なくフローを流すことができます。 逆に、ボトルネックとなる容量を超える流量のフローを流すことはできません。


ここで、少し突飛ではありますが \(s\)-\(t\) カットという概念を紹介します。

\(s\)-\(t\) カットとは、\(G\) の頂点を \(s,t\) が別々になるように \(2\) つの頂点集合に二分することです。 また、\(s\) を含む方を \(A\) , \(t\) を含む方を \(B\) とします。


このとき、\(A\) に属する頂点から \(B\) に属する頂点へ向かう辺の容量の合計をカット容量と呼びます。 頂点を \(A,B\) に二分する方法は複数通り存在しますが、そのうちカット容量が最小になるようなカットを最小カットと呼びます。


例示したカットのカット容量は \(14\) であり、このグラフにおける最小カットです。

ここで、\(s \rightarrow t\) のフローは必ず \(A \rightarrow B\) の移動を経る必要があるので、 最大流はカット容量以下であると言えます。
フローを流す際の流量が選んだ \(s \rightarrow t\) パス上のボトルネックで抑えられることと同様に、 \(s \rightarrow t\) の最大流は \(s\)-\(t\) の最小カットがボトルネックとなります。

よって、\(s \rightarrow t\) の最大流は \(s\)-\(t\) の最小カット容量と一致します。


Dinic 法のライブラリです。\(s \rightarrow t\) の最大流を計算する場面を考えます。

ford-fulkerson 法では容量が正の適当なパスにフローを流すことを繰り返すことで最大流を得ましたが、Dinic 法では容量が正の最短パスにフローを流すことを繰り返します。

基本的な処理を以下に書きます。(頂点集合を \(\mathrm{V}\) , 辺集合を \(\mathrm{E}\) とします)

  1. \(s\) からの BFS によって各頂点までの最短経路を求め、特に \(s \rightarrow t\) の最短経路長を \(D\) とする。
  2. \(s\) から DFS を行い、\(t\) に到達できればフローを流す処理を繰り返す。ただしこのとき、以下の工夫を取り入れる。
    • BFS で得た最短経路を形成する辺のみに注目すると、DFS を行う対象は DAG である。
    • 今いる頂点 \(x\) から出る辺のうち \(i\) 番目: \(e_{x,i}\) を使って \(t\) に到達できなかったならば、今後 \(e_{x,i}\) を見る必要はない。
    • 長さ \(D\) のパスで採用される可能性がない辺を排除し、採用される可能性がある辺だけを見るようにする。各頂点について、何番目の辺まで調べたかをメモしておけば良い。
  3. \(2\) の処理で一度でもフローを流せたなら \(1\) の処理に戻り、そうでなければ処理を終了する。

フローを流す操作を繰り返す。つまり \(2\) 処理を完了するたびに \(s \rightarrow t\) パスの最短経路長は増加していきます。最短経路長は最大で \(|\mathrm{V}|\) なので、\(1,2\) の処理は \(O(|\mathrm{V}|)\) 回行われます。\(1\) の BFS に \(O(|\mathrm{E}|)\) 時間、\(2\) の処理の DFS に \(O(|\mathrm{V}||\mathrm{E}|)\) 時間かかるので、全体で \(O(|\mathrm{V}|{|\mathrm{E}|}^{2})\) 時間となります。計算量だけ見れば遅く思われますが、実際はかなり高速に動作することがほとんどと言われています。どうしても遅い場合は隣接リストの辺の順番をシャッフルすることも有効らしいです。

    - T 型の フロー (Dinic Algorithm)
    - はじめ、全ての頂点対 (u,v) に対して u → v の容量は 0。
    |
    - コンストラクタには頂点数 V を渡す。
        |- 頂点番号は 0 以上 V 未満
    |
    - 辺は edge クラスで表す
        |- メンバは {容量,逆辺の edge クラスへのポインタ} のみ
        |- 別で隣接リスト G があるので、端点をメンバとして持つ必要性は薄い (必要に応じて追加してください)
    |
    - add_edge(u,v,c) := 有向辺 u → v の容量を +c する。(c は 0 以上)
    - path(u,v) := 容量が正の辺のみを通って u→v の最短パスを求める。
                    / 最短パス上の edge のポインタを通った順に並べて返す 
                   / ( 単体使用の前にも bfs(s) してある必要がある。)
                  / t に到達できなかった場合は空集合を返す
    - Flow(u,v,f) := u → v の +++1 本のパス+++ にフローを f 流す。流せなければ false を返す
    - MaxFlow(u,v) := u → v のネットワークにフローを流せるだけ流し、流した流量を返す。
    |
    - 設計&思想 : 
        |- グラフは隣接リスト G を参照する。
        |- dfs で見る辺を枝刈りするため、各頂点に対して次にみるべき頂点をメモしておく。
            |- iter[x] := dfs において頂点 x から、次に参照するべき辺の G[x] における index
            |- backward[x] := x から移動できる辺がなくなったとき、戻るべき頂点
        |- 多重辺を回避するため G とは別に辺集合 E を管理する。
        |- 多重辺を許す必要がある場合は add_edge を編集する。
*/
template<typename T = long long>
class mxfl{
    /*   
        Copyright ©️ (c) okoteiyu 2024. 
        Released under the MIT license(https://opensource.org/licenses/mit-license.php) 
    */
                   

    private : 

    struct edge{
        // u → v の、容量 cap の有向辺
        int u,v;
        T cap;// 容量
        
        edge* r_edge;// 逆辺へのポインタ
        edge(int u_ , int v_ , T cap_):u(u_),v(v_),cap(cap_){}
    };

    // 頂点数
    int V;

    // 特殊シンボル
    const int UNDEFINED_NODE = -1;
    const int UNREACHED = -1; // bfs で到達不可能を表すシンボル
    const T ZERO_EPS = 0;// 容量が 0 であることを示す (double の場合に微小量として活躍)
    
    // グラフ 
    vector<unordered_map<int,edge*>> E;// 辺集合 ( iter(後述) で参照できない )
    vector<vector<pair<int,const edge*>>> G; // 隣接リスト ( iter(後述) で参照できる )
    // [i][j] := 辺 i → j に流れた流量 (逆辺に負の流量を流すので、負になることもある)
    vector<unordered_map<int,T>> F;

    // dfs を行うときに参照する。
    // dfs を行うたびに始点から順に上書きされるので、使いまわして OK
    vector<int> iter; // [x] := 頂点 x にいるとき、次にみる辺の G[x] における index
    vector<int> backward;// [x] := x から移動できる辺がなくなったとき、戻るべき頂点
    
    // 何度も bfs するので 配列を使い回す。
    vector<int> dist;// 最後に bfs をはじめた頂点からの最短経路長

    public:
    mxfl(int V_)
    :V(V_),E(V_),G(V_),F(V_)
    ,dist(V_,UNREACHED),iter(V_,0),backward(V_,UNDEFINED_NODE){}
    ~mxfl(){
        for(int i = 0 ; i < V ; i++){
            for(pair<int,edge*> e: E[i]){
                // edge を 解放すると r_edge も解放される
                if(i < e.first)delete e.second;
            }
        }
    }

    // u → v の容量を c 増やす
    void add_edge(int u , int v , T c){
        assert(u != v);
        // 多重辺をマージ
        if(E[u][v] != nullptr && E[v][u] != nullptr){
            E[u][v]->cap += c;
            return;
        }
        assert(E[u][v] == nullptr && E[v][u] == nullptr);
        E[u][v] = new edge(u,v,c);
        E[v][u] = new edge(v,u,0);
        E[u][v]->r_edge = E[v][u];
        E[v][u]->r_edge = E[u][v];

        G[u].push_back({v,E[u][v]});
        G[v].push_back({u,E[v][u]});
    }

    // s から 各ノード への最短経路長を計算し、ついでに iter を初期化する
    void bfs(int s){
        vector<int>(V,UNREACHED).swap(dist);
        vector<int>(V,0).swap(iter);
        queue<int> q;q.push(s);
        dist[s] = 0;
        while(!q.empty()){
            int now = q.front();
            q.pop();
            for(pair<int,const edge*> e : G[now]){
                if(e.second == nullptr)continue;
                if(dist[e.first] != UNREACHED)continue;
                if(e.second->cap <= ZERO_EPS )continue;
                dist[e.first] = dist[now]+1;
                q.push(e.first);
            }
        }
    }

    // s → t 最短経路上の edge クラスのポインタを通った順に返す
    // 単体使用の前にも bfs(s) してある必要がある。
    // t に到達できなかった場合は空集合を返す
    vector<edge*> path(int s , int t){
        assert(s!=t);
        vector<edge*> res(0);
        int now = s;
        
        // s からの最短路に含まれる辺のみに注目したグラフは DAG であることを利用している
        while(now != UNDEFINED_NODE){
            if(now == t)break;

            // 移動先を見つけるまで iter[now] をインクリメント
            for(;iter[now] < G[now].size(); iter[now]++ ){
                pair<int,const edge*> e = G[now][iter[now]];
                if(e.second == nullptr)continue;
                if(e.second->cap <= ZERO_EPS)continue;
                if(dist[e.first] <= dist[now])continue;// 最短路のみ注目
                break;// 見つけたら終了
            }

            // 見る辺が見つからなかった場合一つ前に戻る。
            if(G[now].size() <= iter[now]){
                now = backward[now];
                if(res.size() != 0)res.pop_back();// 直前に使った辺を不採用
                // iter[now] を使うと t に到達できないことがわかったので、今後 iter[now] は不採用
                if(now != UNDEFINED_NODE)iter[now]++;// 戻った先の見るべき辺を 1 つずらす
            }else{
                pair<int,const edge*> e = G[now][iter[now]];
                backward[e.first] = now;// 戻り先を新たに設定
                res.push_back(const_cast<edge*>(e.second));// 使った辺を追加
                now = e.first;// 移動する
            }
        }
        return res;
    }



    // 実際に s - t パスに f 流す
    // 流せないなら流さずに false を返す
    bool Flow(int s , int t , T f){
        bfs(s);
        if(dist[t] == UNREACHED)false;
        vector<edge*> p = path(s,t);
        if(p.size()==0)return false;
        T bottle_neck = p.back()->cap;
        for(const edge* e : p)bottle_neck = min(bottle_neck,e->cap);
        if(bottle_neck < f)return false;
        for(edge* e : p){
            F[e->u][e->v] += f;
            e->cap -= f;
            e->r_edge->cap += f;
            F[e->v][e->u] -= f;// 逆辺には負の流量が流れたことにする
        }
        return true;
    }

    // s → t に流せるだけ流す。流した流量を返す
    T MaxFlow(int s , int t){
        T res = 0;
        vector<edge*> p;
        while(true){
            bfs(s);
            if(dist[t] == UNREACHED)break;
            while((p = path(s,t)).size() != 0){
                T bottle_neck = p.back()->cap;
                for(const edge* e : p)bottle_neck = min(bottle_neck,e->cap);
                res += bottle_neck;
                for(edge* e : p){
                    F[e->u][e->v] += bottle_neck;
                    e->cap -= bottle_neck;
                    e->r_edge->cap += bottle_neck;
                    F[e->v][e->u] -= bottle_neck;// 逆辺には負の流量が流れたことにする
                }
            }
        }
        return res;
    }

    vector<edge> edges(){
        vector<edge> res;
        for(int x = 0 ; x < V ; x++)for(pair<int,T> e : F[x])res.emplace_back(x,e.first,e.second);
        return res;
    }

};
