木構造

木構造

木は以下を満たす、グラフの特殊なインスタンスです。

  • 辺の本数が頂点数より \(1\) 少ない。
  • 全ての頂点は連結である。
また、全ての連結成分が木であるグラフを森と言います。

根付き木

ある頂点を根とすることで、木構造及び木構造上での処理の見通しを良くします。頂点 \(0\) を根とすると、先ほど提示した木は以下のような構造になります。

「どこが根やねん!!!」というツッコミはおいておいて、根付き木の説明します。

  • 頂点 \(u\) の親 : \(u\) に隣接する頂点であり、\(u\) より根に近いもの (上側の頂点)
  • 頂点 \(u\) の子 : \(u\) に隣接する頂点であり、\(u\) より根から遠いもの (下側の頂点)
  • 頂点 \(u\) の祖先 : \(u\) から親をたどって (上移動のみで) 到達できる頂点
  • 頂点 \(u\) の深さ : 根から \(u\) までのパスの長さ
  • 根は親を持たない。
  • 子を持たない頂点を葉と言う。

LCA (Lowest Common Ancestor)

異なる \(2\) つの頂点 \(u,v\) の LCA とは、\(u,v\) の共通の祖先のうち根から最も遠いもの (\(u,v\) から最も近いもの) です。

求め方

木の辺を根方向に向き付けすると、木は Functional Graph になります。根からは根にリンクさせておいてください。

Functional Graph では状態の遷移をダブリングで管理することができます。具体的には、以下のテーブルを構築します。

  • \(\mathrm{parent}[k][v] := \) 頂点 \(v\) の \(2^k\) 個上の祖先
このダブリングテーブルを用いて以下の \(2\) ステップを行い、LCA を求めます。

深さを合わせる。

頂点 \(u,v\) の LCA の計算の際、\(u\) の深さは \(v\) の深さ以上とします。そうでない場合は \(u,v\) をスワップすれば良いです。この時、頂点 \(u\) が頂点 \(v\) と同じ深さになるまで \(u\) を親方向に移動させます。ここで \(u\) の祖先は、\(v\) と深さの大小関係が 以下/より大きい の \(2\) つの領域に分断されるので、それらの領域の境界を見つけるために二分探索を行います。

ここで行う二分探索は少し特殊で、自分はダブリング二分探索と呼んでいます。ダブリングテーブルで、各頂点の \(2\) の冪乗個上の祖先が何かはわかっています。よって、\(2\) の冪乗 \(2^k\) が大きい方から「\(u\) の \(2^k\) 個上の祖先が \(v\) の深さ以下であればその頂点に移動する」を繰り返すことで、頂点 \(u,v\) の深さが同じになるように \(u\) を移動させることができます。これは \(10\) 進数表示の整数を \(2\) 進数表示に変換する際の手法と同じです。

この時点で \(u=v\) ならば、LCA は \(u\) です。

共通祖先を調べる

\(u,v\) の深さは同じなので、共通祖先から \(u,v\) までの距離はどちらも等しいです。\(\mathrm{same}(x) := \)「\(u\) の \(x\) 個上の祖先が \(v\) の \(x\) 個上の祖先と等しいか」という判定は、Yes/No の \(2\) つの領域に分割されるので、これもダブリング二分探索で解くことができます。

\(2\) の冪乗 \(2^k\) が大きい方から、\(\mathrm{same}\) の判定が No なら \(u,v\) をそれぞれ \(2^k\) 個上に移動することを繰り返すことで、\(u,v\) が境界まで移動し、\(u,v\) は共通祖先の一つ下の頂点になります。よって、この状態の \(u\) の親頂点が LCA です。

木圧縮 (Auxilirary Tree)

頂点数が \(N\) の木の頂点集合の部分集合 \(S\) に対して、\(S\) に含まれる頂点をカバーする部分グラフのうち、頂点数が最小の木を考えます。

この部分グラフの頂点数は \(O(N)\) 個です。ここで、余計な一本道を圧縮することで頂点数が \(O(|S|)\) 個の木を新たに構築します。

このようにして構築される木を、元の木に対して Auxiliary Tree と言います。Auxiliary Tree は \(S\) に含まれる頂点に加えて、\(S\) の頂点間の位置関係を補助する頂点も含みます。

アルゴリズム

\(S\) の頂点を、元の木の根からの DFS で初めて到達するタイミングが早い順に並べ替えます。\(S = \{2,3,13,4,5,15\}\) の場合、\(S = \{2,4,3,5,15,13\}\) が一例です (DFS の道順はいくつもあるので、そのうちなら何でも良い)。

また実装の共通化のため、\(S\) の末尾には番兵として \(S_0\) を追加しておきます。以降、\(S = \{2,4,3,5,15,13,2\}\) とします。

DFS の到達順に頂点を調べながら Auxiliary Tree を構築します。最後に調べた頂点を \(r\) とし、次の処理で注目する頂点の集合を以下で定義します。

  • \(\mathrm{frontline} := \) 「Auxiliary Tree に含まれると確定している頂点」で \(r\) の最も遠い祖先から \(r\) までのパス上の「Auxiliary Tree に含まれると確定している頂点」を根から近い順に並べたもの。
補足として、Auxiliary Tree に含まれる頂点は、\(S\) に属する頂点及び \(S\) に含まれる \(2\) 頂点の LCA になりうる頂点です。Auxiliary Tree に含まれる頂点がどんなものかはわかっているのですが、\(S\) に属する \(2\) 頂点の組合せが \(O(|S|^2)\) 個なので、初めから全部列挙しておくことはできず、Auxiliary Tree の構築中に確定させていく必要があります。

Auxiliary Tree の構築で行うべきことは、以下の \(3\) 点です。

  • \(\mathrm{frontline}\) を 構築/変更 する
  • Auxiliary Tree に含まれる頂点を確定させていく
  • 確定した頂点たちに辺を張っていく (元の木のパスを圧縮していく)
これを踏まえてアルゴリズムの様子を図で見てみましょう。黒の破線で囲まれた領域のうち「Auxiliary Tree に含まれると確定している頂点」が \(\mathrm{frontline}\) です。はじめ、初期値として \(S_0\) のみが \(\mathrm{frontline}\) に含まれます。

まず頂点 \(S_i=4\) を調べます。ここで、\(\mathrm{frontline}\) の最も下の頂点 (\(r\)) と \(S_i\) の LCA は Auxiliary Tree に含まれる頂点として確定します。そして \(\mathrm{frontline}\) が変更され、頂点 \(2\) が \(\mathrm{frontline}\) から除外されます。

このとき、深さが \(\mathrm{LCA}(r,S_i)\) よりも深い頂点をすべて \(\mathrm{frontline}\) から除外します。\(\mathrm{frontline}\) から除外されることで、今後の構築で注目されることはなくなり、Auxiliary Tree での位置や隣接する頂点が確定しました。

深い順に \(\mathrm{frontline}\) から除外し、除外順で隣接する頂点同士に辺を構築します。また最後に除外された頂点が \(\mathrm{LCA}(r,S_i)\) でないなら、その頂点と \(\mathrm{LCA}(r,S_i)\) の間にも辺を構築します。今回は \(2\) のみが除外され、\(2\) は \( \mathrm{LCA}(S_i,r) = 7\) ではないので、\(2\) と \(7\) の間に辺を構築します。

新たに頂点 \(\mathrm{LCA}(r,S_i)\) が確定したので \(\mathrm{frontline}\) に \(7\) を追加し、その後今調べた頂点 \(S_i = 4\) も \(\mathrm{frontline}\) に追加します。

補足 : このアルゴリズムのもとでは、\(\mathrm{frontline}\) に格納される頂点の深さが単調になるので、\(\mathrm{frontline}\) は末尾に追加/削除ができれば良く、stack で実現できます。

次は \(S_i = 3\) を調べます。\(\mathrm{LCA}(S_i,r) = \mathrm{LCA}(3,4) = 4\) です。このとき \(\mathrm{frontline}\) からは \(4\) が除外された後、手続きに従い \( \mathrm{LCA}(S_i,r) = 4\) と \(S_i = 3\) が追加されます。除外された頂点が \( \mathrm{LCA}(S_i,r)\) と等しいので、辺の構築は行いません。

次に \(S_i = 5\) を調べます。\( \mathrm{LCA}(S_i,r) = 7\) 以下の頂点が \(\mathrm{frontline}\) に存在するので、それら (\(\{3,4,7\}\)) を除外しながら辺を構築します。最後に除外された頂点が \( \mathrm{LCA}(S_i,r) = 7\) なので、\(7,7\) に辺を構築することはできません。

その後、今調べた \(7,5\) を順に \(\mathrm{frontline}\) に追加します。

以降、同じ具合に構築を進めます。

\(S_i = 15\) を調べると、\(5\) のみが \(\mathrm{frontline}\) から除外され、\(5\) は \(\mathrm{LCA}(S_i,r) = 14\) ではないので、\(5\) と \(14\) の間に辺を構築します。


\(S_i = 13\) を調べると、\(15,14,7\) が \(\mathrm{frontline}\) から除外され、\(15,14\) と \(14,7\) の間に辺が構築されます。また最後に除外された \(7\) は \(\mathrm{LCA}(S_i,r) = 0\) ではないので、\(7\) と \(0\) の間に辺を構築します。


最後に、番兵として入れておいた \(S_0 = 2\) をもう一度調べます。このとき \(\mathrm{LCA}(S_i,r) = 0\) は Auxiliary Tree 全体の根です。よって、\(S_0\) を調べることで \(\mathrm{frontline}\) に残っている頂点/辺を全て確定して処理を完了させることができます。\(\mathrm{frontline}\) からは \(13,0\) を除外して \(13,0\) の間に辺を構築します。


この時点で確定していない辺や頂点は全て除外して OK です。木圧縮ができました。

木の走査に \(O(|S|)\) 時間、その中での \(\mathrm{LCA}\) のクエリごとに \(O(\log{N})\) 時間かかり、DFS 到達順でのソートに \(O(|S|\log{|S|})\) 時間かかるので、合わせて \(O(|S|(\log{|S|} + \log{N}))\) 時間かかります。なお LCA は別の方法で前処理すると、でクエリあたり定数時間になります。

    static な根付き木の基本機能。コンストラクタは (頂点数 , 根頂点 , 頂点の値の初期値)。
        |- 頂点番号は 0 から m_N-1
        |- reroot(r) := 根を r にして構築 ( O(m_N) 時間かかる)
    - 辺重みや頂点重みを持つ。
        |- G[x] := 頂点 x に隣接する Edge オブジェクトの集合
    |
    - add_edge で辺を追加している間は m_isStatic が false。
        |- lca(), dist() などのクエリを処理する際に、自動で m_isStatic が true になり、木が static な状態になる。
        |- m_isStatic が false　の状態で lca() , dist() などが呼ばれる際に** 1 度だけ ** 必要なデータを計算する(m_parent など)
    |
    - AuxiliaryTree(vector<int> S)
        |- 頂点の部分集合 S をカバーする木圧縮。計算量は 0(|S|)。
        |- 返り値は pair<vector<int> , unordered_map<int,vector<int>>> のペア
            |- 補助的な頂点を含めた頂点集合 V と、unordered_map を使った隣接リストを pair で返す。
            |- 返り値の隣接リストは、Edge ではなく頂点番号の集合
        |- V は DFS 到達順でソートずみ
    |
    - dist(x) : m_root からの距離(辺重みの総和) (到達不可能なら -1)
    - dist(int u ,int v) : u-v の距離(辺重みの総和) (到達不可能なら -1)
    - parent(x) : x の親
    - dfs() : m_root からの dfs 到達順に頂点を並べた vector
    - dfs_order() : 各頂点について、dfs 到達順(0-index)をメモした vector を返す
    - lca(u,v) : u,v の LCA
    - [x] := 頂点 x に隣接する頂点の集合 (vector) への参照
    - いずれも、未定義のものは -1 を返す
    |               
    - なぜ基本機能に Auxiliary Tree が含まれるのか
        -> LCA を使うから
            |- LCA や 根からの dist を使うやつは全部ここに書く
    - 直径の計算は？
        - 最初に決めた根からの dist が最大の頂点を x として、construct(x) で根を x に変更
            -> x から最長の頂点が答え
*/
template<typename type_Nodeweight , typename type_Edgeweight>
class StaticTree{
    /*   
        Copyright ©️ (c) okoteiyu 2024. 
        Released under the MIT license(https://opensource.org/licenses/mit-license.php) 
    */

    public:
    struct Edge{
        int from, to; // 両端の頂点番号 (無向の場合も有向と捉える)
        type_Edgeweight weight;
        int id;// 辺にも id を割り振る場合がある
        Edge(int f,int t,type_Edgeweight w):from(f),to(t),weight(w){}
    };
    private:
    
    int m_N;// 頂点数 (番号は 0~m_N-1)
    int m_root = 0;// 計算を行うための暫定的な根
    vector<vector<Edge>> m_G;// 隣接リスト
    vector<type_Nodeweight> m_nodeVal;// 頂点が持つ値

    vector<int> m_dist;// m_root からのパスの長さ
    vector<type_Edgeweight> m_weightsum;// m_root からのパス上の辺の重みの総和
    vector<int> m_dfs;// dfs 到達順に頂点を並べたもの
    vector<int> m_dfsOrder;// [x] := 頂点 x の dfs 到達順
    
    vector<vector<int> > m_parent;// [i][x] := 頂点 x の 2^i 個上の親頂点 (なければ -1)

    // LCA とかを求めた後は static な状態になるので、add_edge できない(ということを持つフラグ)
    bool m_isStatic = false;// static な状態、つまり init 済みかどうか

    // static ではないときに呼ばれたら、一度だけ LCA などに必要なデータを計算して、static な状態になる
    void init(){
        if(m_isStatic)return;// すでに static なら、必要データは計算済み
        m_isStatic = true;
        stack<int> s;
        s.push(m_root);
        m_dist[m_root] = 0;
        m_weightsum[m_root] = 0;
        int px = 0;
        while(!s.empty()){
            int now = s.top();
            s.pop();
            m_dfs.push_back(now);
            for(Edge e : m_G[now]){
                if(m_dist[e.to] != -1)continue;
                m_weightsum[e.to] = m_weightsum[now] + e.weight;
                m_dist[e.to] = m_dist[now] + 1;
                s.push(e.to);
                m_parent[0][e.to] = now;
            }
        }
        for(int i = 0 ; i < m_dfs.size() ; i++)m_dfsOrder[m_dfs[i]] = i;
        
        for(int i = 1 ; i < m_parent.size() ; i++){
            for(int x = 0 ; x < m_N ; x++){
                if(m_parent[i-1][x] == -1)m_parent[i][x] = -1;
                else m_parent[i][x] = m_parent[i-1][m_parent[i-1][x]];
            }   
        }
    }

    public:

    StaticTree(){}
    StaticTree(int N_ , int r , type_Nodeweight init_v){
        m_N = N_;
        m_G = vector<vector<Edge>>(m_N);
        m_nodeVal = vector<type_Nodeweight>(m_N,init_v);
        reroot(r);
    }


    // 根を r に変更する (O(N)時間)
    void reroot(int r){
        // メンバを初期化するだけ。メンバの計算は、必要になった際に init() で一度だけ行われる
        m_isStatic = false;
        m_root = r;
        m_dist = vector<int>(m_N,-1);
        m_weightsum = vector<type_Edgeweight>(m_N,-1);
        m_parent = vector<vector<int>>(log2(m_N)+2,vector<int>(m_N,-1));
        m_dfs = vector<int>(0);
        m_dfsOrder = vector<int>(m_N,-1);
    }


    // u-v に無向辺を追加 (多重になったり、2-連結にならないように注意)
    void add_edge(int u , int v , type_Edgeweight w = type_Edgeweight(1)){
        assert(!m_isStatic);// static な状態になったら add_edge できない
        m_G[u].push_back(Edge(u,v,w));
        m_G[v].push_back(Edge(v,u,w));  
    }

    // 頂点 u の持つ値を x にする
    void update_val(int u , type_Nodeweight x){
        assert(!m_isStatic);// 現状 static でなくても良いが、念の為
        assert(0 <= u && u <= m_N);
        m_nodeVal[u] = x;
    }

    // 根頂点の番号
    int root(){return m_root;}

    // 頂点の値
    type_Nodeweight node_v(int u){return m_nodeVal[u];}

    // 頂点数
    int size(){return m_N;}
    
    // root() からの距離
    int dist(int x){
        assert(0<=x&&x<m_N);
        init();// static じゃないタイミングに、1 度だけ呼ばれる
        assert(m_dist[x] >= 0);// 到達可能を保証
        return m_dist[x];
    }
    // root() からのパス上の辺の重みの総和 
    type_Edgeweight path_weight(int x){
        assert(0<=x&&x<m_N);
        init();
        assert(dist(x) >= 0);// 到達可能を保証
        return m_weightsum[x];
    }
    // u-v の距離 
    int dist(int u , int v){
        assert(0<=u&&u<m_N && 0 <= v && v < m_N);
        init();
        assert(dist(u)>= 0 && dist(v) >= 0);// 到達可能を保証
        return dist(u)+dist(v)-2*dist(lca(u,v));
    }

    // u-v パス上の辺の重みの総和 
    type_Edgeweight path_weight(int u ,int v){
        assert(0<=u&&u<m_N && 0 <= v && v < m_N);
        init();
        assert(dist(u,v)>=0);
        return path_weight(u)+path_weight(v)-2*path_weight(lca(u,v));
    }

    // root() を根とする場合の親頂点 (存在しなければ -1)
    int parent(int x){
        assert(0<=x&&x<m_N);
        init();// static じゃないタイミングに、1 度だけ呼ばれる
        return m_parent[0][x];
    }

    // root() からの dfs 到達順に頂点番号を返す
    vector<int> dfs(){
        init();// static じゃないタイミングに、1 度だけ呼ばれる
        return m_dfs;
    }

    // 各頂点について、root() からの dfs 到達順で何番目かをメモした配列を返す(root() から到達不可能なら -1 とする)
    vector<int> dfs_order(){
        init();// static じゃないタイミングに、1 度だけ呼ばれる
        return m_dfsOrder;
    }


    // x に隣接する頂点の vector (参照) を返す (_static じゃなくても呼び出せる)
    // これによって、普通の隣接リストのように扱える
    const vector<Edge>& operator [](int x){
        assert(0 <= x && x < m_N);
        return m_G[x];
    }

    // u,v の LCA
    int lca(int u, int v) {
        assert(0<=u&&u<m_N && 0 <= v && v < m_N);
        init();// static じゃないタイミングに、1 度だけ呼ばれる
        if(m_dist[u] < 0 || m_dist[v] < 0)return -1;
        if (m_dist[u] < m_dist[v]) swap(u, v); // dist(u) >= dist(v) となるようにする
        int K = m_parent.size();
        for (int k = 0; k < K; k++) {
            if((m_dist[u] - m_dist[v])&(1<<k))u = m_parent[k][u];
        }
        if (u == v) return u;
        for (int k = K - 1; k >= 0; k--) {
            if (m_parent[k][u] != m_parent[k][v]) {
                u = m_parent[k][u];
                v = m_parent[k][v];
            }
        }
        return m_parent[0][u];
    }

    // r を根とした際の、u,v の LCA (不思議な論理式でもとまる)
    int lca(int r , int u , int v){return lca(r,u)^lca(r,v)^lca(u,v);}

    // 頂点集合 S に含まれる頂点を、dfs 到達順でソートする
    template<class Integer>
    void sort_dfs_order(vector<Integer>& S){sort(S.begin(),S.end() , [&](int u , int v){return m_dfsOrder[u] < m_dfsOrder[v];});}

    /*
        (要素がユニークな) 頂点集合 S をカバーする最小の部分グラフにおいて、一本道のパス上の頂点を全て圧縮した木を作る。
        - 圧縮後の木は隣接リストで表現 : unordered_map res[x] := x に隣接する頂点 (Edge ではなく隣接する頂点番号)
        - 返り値は {新たな頂点集合 , 隣接リスト} とする。新たな頂点集合は、元の木の root() からの dfs 到達順でソートされたもの。
            |- 先頭の要素が、圧縮された木の頂点で、root() に最も近い (最も祖先)
        - int , long long どっちも受け取れるように、テンプレートで抽象化する。ただし内部実装は int 
    */
    template<class Integer>
    pair<vector<Integer> , unordered_map<Integer,vector<Integer>>> AuxiliaryTree(vector<Integer> S){
        assert(S.size() != 0);
        init();// static じゃないタイミングに、1 度だけ呼ばれる
        
        unordered_set<Integer> subnodes;// 圧縮後の木の頂点集合 (重複を回避するための set)
        unordered_map<Integer,vector<Integer>> res;
        
        // m_dfs 到達順に調べて構築していく
        sort_dfs_order(S);

        // これまでに見た最も祖先から、最後に見た頂点までにパス上の (採用が確定している) 頂点
        vector<int> frontline(1,S.front());
        S.push_back(S.front());// 番兵 (圧縮後の木の根以下の未確定の辺を最後に確定させるため)

        for(int i = 1 ; i < S.size(); i++){
            int LCA = this->lca(S[i-1],S[i]);
            // 頂点集合に追加。
            subnodes.insert(S[i]);//番兵があるので S の要素は過不足なく全部含まれる。
            subnodes.insert(LCA);
            if(LCA != S[i-1]){// 最後に見た頂点が LCA にならないなら辺の確定を行う
                assert(frontline.size() != 0);
                // stack のうち LCA 以下の辺を確定させる
                while (frontline.size() != 0){
                    if(m_dist[LCA] > m_dist[frontline.back()])break;// LCA 以下の頂点だけ見る
                    int par = LCA;// frontline.back() の親頂点候補
                    if(frontline.size() >= 2){
                        if(m_dist[frontline[frontline.size()-2]] >= m_dist[LCA])par = frontline[frontline.size()-2];
                    }
                    if(frontline.back()!=par){// 辺が確定
                        res[par].push_back(frontline.back());
                        res[frontline.back()].push_back(par);
                    }
                    frontline.pop_back();
                }
                frontline.push_back(LCA);
            }
            frontline.push_back(S[i]);
        }
        vector<Integer> nV;// 圧縮後の頂点集合
        for(int x : subnodes)nV.push_back(x);
        sort_dfs_order(nV);// 新たな頂点集合を m_dfs 到達順にソート
        return {nV,res};
    }

};
