てゆ掲示板 Wiki

全体構成

okoteiyu.com は、Github Pages のみを用いて公開しています。つまり、okoteiyu.com にはバックエンドが存在せず、全てのコンテンツをフロントに用意しておく必要があります。
全てのコンテンツを愚直に HTML に埋め込んで表示するのはあまりにも非効率的なので、てゆ掲示板では、HTML を抽象化されたフレームとして扱います。それぞれのコンテンツは形式的なデータとして html 要素に埋め込まれ、JavaScript がそれらのデータからページを構築する流れです。 
======= index.html =======
|## フロント操作&リソース　  |    
|コンテンツリストを定義.     |    
|    → JS がデータを構築　  |         
|    → HTML、ビューに反映　 |           ========== TimeLineAPI.js ===========
==========================    <=>    |## バックエンドに対応する立ち位置       |
          |                          |index.html に定義されたデータや操作から |  
          |                          |ページ情報(配置、イベント)を構築する。   |
          |コンテンツリストに           =====================================
          |リンク情報を埋込み                   |
          |                                  |index.html から取得したデータを
          |                                  |統合して、URL パラメータ付きリンクにする
========== articles ======                   |
|## 記事データを格納        |              ======= article_constructor.js =====   
==========================     <=>      |## フロント処理に対応              　 |
                                        |TimeLineAPI から URL パラメータとして |
                                        |受け取った情報を元に記事データを構築する。|
                                        ====================================
具体的にはどんなデータがどうやって定義されるの？
いい質問でし。まず、index.html ページには以下の要素があるでし。
  • ポスト一覧
  • 投稿者一覧
  • フィルター
こいつらのデータを HTML に直接埋め込むのは泥臭いから、なるべく抽象化しようとしたというのがさっきの話なのね。
まずはじめにフィルター要素ですけども、ここで抽象化されているのはカテゴリ指定の種類なのね。カテゴリの種類は投稿者と一対一対応しておる。ここではまず投稿者データを取得して、その投稿者の担当カテゴリをフィルターのドロップダウンリストに追加することにしておる。 
<!-- 投稿者データアイテム -->
<div class = "person_data" data-user-name = "顔面惑星" data-post-category = "日常"
    data-icon-link = "./static/image/face_planet.jpg">顔面惑星はいつも上から見てるよ(🍥⩌ ︿ ⩌🍥)
</div>
こいつのデータは、index.html ページでのサムネイルやカテゴリに加えて、今見ている文章のアイコンにもなるヨット。様式は見ての通りﾃﾞｼﾘｯﾄﾙ。
なるほど、確かに投稿者データの HTML 要素を各所に散りばめるより圧倒的に綺麗だね！
うむ。では次にポスト/記事データについて説明するよ。まず、index.html には記事データは直接埋め込んでいないよ!!! index.html には記事ビューの iframe だけ用意して、index ページ上で選択した記事をビューに表示しているのラ。
記事データは web ページであれば何でも良いけれど、特に articles フォルダ配下のものは特別に ポスト ID と関連づけるようにしているよ。そしてポスト/記事を構築するため、index.html にポスト/記事データを以下の様式で定義しているよ。 
<!-- id はポストリンクの自動割り当て, link は明示的なリンク -->
<!-- link と id が両方指定されているなら、link を採用. -->

<div class = "post_data" data-post-category = "" 
data-post-time = "2000/01/06 00:00:00" data-article-link=""
data-post-priority = "0" data-post-thumbnail = ""
data-post-content = "おこていゆ誕生"></div>

<div class = "post_data" data-post-category = "技術/運営" 
data-post-time = "2026/05/08 01:36:00" 
data-post-id = "T0000" data-post-priority = "1" data-post-thumbnail = ""
data-post-content = "てゆ掲示板について"></div>
これは少し説明が必要かもね。(かーもね!) 以下のデータを定義しているよ。あと少し事前説明をすると、ポストってのは index.html に表示されるもので、記事ってのはこのページみたいに、ポストと関連づけられてるものだよ。記事が index.html のビューに表示されるのねー。
  • data-post-category : 投稿者の担当カテゴリに対応するよ。このデータを元に、ポストや記事のアイコンが決定されるよ。この記事だと、技術/運営 カテゴリだからオイラが担当しているのね。ちなみに、どの投稿者のカテゴリにも合致しない場合、デフォルトとして設置された投稿者の投稿になるよ！
  • data-post-time : YYYY/MM/DD hh:mm:ss 形式で定めた日時データだよ。time.js で処理可能なデータ形式に合わせておる。
  • data-article-link : ビューに表示する記事内容のリンクだよ。ここが指定されていたら、ポスト ID による自動割り当ては OFF になるよ。
  • data-post-id : ポストの ID だよ。リンクが指定されていなければ、リンクに自動割り当てされて articles/[ポスト ID]/page.html が記事内容になるよ。リンクも ID も指定がなければ記事と関連づかないポストになるよ!! あと、リンクが指定されている場合でも ID はあったほうがいいヨット。特定の記事を開いた状態でページ URL を共有する際の URL パラメータになるからネット！
  • data-post-priority : 優先ポストに指定するかどうかを決めるよ!! 0 なら通常ポスト、1 なら優先ポストとして優先ポスト一覧に追加されるよ！
  • data-post-thumbnail : ポスト表示の時に、ここで指定した画像をサムネイルにできるよ。なければサムネイルは無しになるよ！
  • data-post-content : ポスト表示の際に表示されるテキストだよ。
うーん、さっきより少し複雑になってけど、それでも泥臭く埋め込むよりはだいぶ綺麗だね。
そうなのサ。最後に、記事ページに渡されるデータについて説明するよ。TimeLineAPI.js は、ポスト要素に記事へのリンクを付与するんだけど、この時データを以下の通り URL パラメータとして渡しているんだ。 
/board/articles/T0001/page.html?icon_src=./static/image/oko_icon.jpg&user_name=おこていゆ&category=技術/運営
まあパラメータの内容は見たまんまで、このデータを元に、記事データ内のフレームにデータが割り当てられるよ。実際、この記事の HTML にはカテゴリ名もアイコンのソースもひとつも埋め込まれていないよ！
